# 04_COMMON_LAYER

## Purpose

The common layer is the durable asset that remains useful even when models, tools, and interfaces change.

The model is rented.  
The common layer is owned.

## Common layer components

### raw/

Raw sources:

- transcripts;
- articles;
- notes;
- documents;
- exported conversations;
- examples;
- references;
- audio notes;
- screenshots;
- repository snapshots.

### knowledge/processed/

Processed knowledge:

- summaries;
- extracted frameworks;
- concepts;
- decisions;
- source maps;
- definitions;
- comparisons;
- gotchas.

### skills/

Reusable workflows.

### evals/

Tests, rubrics, checklists, acceptance criteria.

### projects/

Active project-specific specs and logs.

### scripts/

Deterministic tools and checks.

## Ingest rule

Raw material should not be treated as knowledge until processed.

A good ingestion extracts:

- source;
- date;
- context;
- claims;
- key ideas;
- uncertainty;
- reusable concepts;
- links to raw material;
- possible applications;
- verification needs.

## Knowledge rule

Processed knowledge must separate:

- fact;
- quote;
- paraphrase;
- inference;
- user preference;
- decision;
- open question.

## Maintenance rule

The common layer must improve over time.

Repeated confusion should become structure.

Repeated correction should become a rule.

Repeated workflow should become a skill.

Repeated verification should become a checklist or script.
