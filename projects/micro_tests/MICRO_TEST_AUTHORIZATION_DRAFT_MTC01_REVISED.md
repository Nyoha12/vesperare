# MICRO_TEST_AUTHORIZATION_DRAFT_MTC01_REVISED

## Statut

Version révisée du brouillon d’autorisation pour MTC01.

Ce document n’autorise toujours pas le test.

Il intègre la revue agent-style `AGENT_REVIEW_MTC01_AUTHORIZATION_DRAFT`.

---

# 1. Micro-test candidat

```text
MTC01 — Sub soutient didgeridoo sans le masquer
```

Question unique :

```text
Un sub peut-il soutenir le corps sans masquer les formants, le souffle et la pression organique du didgeridoo live ?
```

---

# 2. Objet central du test

L’objet central n’est pas le sub.

L’objet central est :

```text
la lisibilité et l’agency du didgeridoo live en présence d’un soutien grave.
```

Le sub est seulement la variable d’accompagnement.

---

# 3. Ce que le test pourrait observer plus tard

Si autorisé un jour, le test devrait observer :

```text
1. le didgeridoo reste-t-il lisible ?
2. le souffle reste-t-il perceptible ?
3. les formants restent-ils vivants ?
4. la pression organique est-elle soutenue ou remplacée ?
5. le sub peut-il se réduire ou se retirer ?
6. Yohan reste-t-il dans le jeu ou sort-il en écoute technique ?
```

---

# 4. Ce qui serait testé

Seulement :

```text
la relation soutien / masquage entre didgeridoo live et fonction sub.
```

Observation attendue :

```text
le sub aide ;
le sub masque ;
le sub remplace ;
le résultat est ambigu ;
le test est mal posé.
```

---

# 5. Ce qui ne serait pas testé

Ne pas tester :

```text
architecture du futur système ;
engine sub final ;
patch Max for Live ;
mapping de performance ;
interface finale ;
scène complète ;
mix complet ;
relation aux guimbardes ;
relation au gong ;
relation au tambour ;
retour au corps global ;
spec Vesperare.
```

---

# 6. Conditions minimales avant autorisation

Le test ne pourrait être autorisé que si :

```text
1. Yohan confirme explicitement qu’il veut essayer ce micro-test ;
2. une seule question est maintenue ;
3. le didgeridoo live reste central ;
4. le sub reste accompagnement ;
5. le test inclut au moins un moment de retrait / réduction du sub ;
6. la charge mentale de Yohan est observée ;
7. le résultat attendu est une observation, pas un son final ;
8. aucun mapping durable n’est créé ;
9. aucun patch n’est conservé comme base ;
10. le résultat positif est traité comme indice local, pas validation du sub engine.
```

---

# 7. Critères d’arrêt

Arrêter immédiatement si :

```text
le test devient un patch ;
il exige une architecture ;
il commence à définir un engine ;
il ajoute les guimbardes, gong ou tambour ;
il ajoute une interface ;
il cherche un beau son final ;
il masque le didgeridoo ;
il empêche Yohan de rester dans le jeu ;
il donne envie de conserver le montage comme base ;
il produit plus de décisions que d’observations.
```
