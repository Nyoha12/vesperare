# SKILL_QUALITY_CHECKLIST

## Purpose

Use this checklist before accepting a new skill.

A skill is not a prompt.  
A skill is a reusable workflow with verification and gotchas.

## Checklist

- [ ] The skill has a clear use case.
- [ ] The skill says when to use it.
- [ ] The skill says when not to use it.
- [ ] Inputs are clear.
- [ ] Steps are ordered.
- [ ] Output format is defined.
- [ ] Verification is defined.
- [ ] Gotchas are included.
- [ ] Examples are included or planned.
- [ ] Related files are listed if useful.
- [ ] Scripts or references are used when deterministic help is possible.
- [ ] The skill was derived from a real workflow or clear repeated need.
- [ ] The skill is small enough to compose with other skills.

## Failure conditions

Reject the skill if:

- it is just a long prompt;
- it tries to do too many things;
- it has no verification;
- it was created abstractly without a real use case;
- it duplicates another skill.

## Output

PASS / NEEDS REVISION / REJECT
