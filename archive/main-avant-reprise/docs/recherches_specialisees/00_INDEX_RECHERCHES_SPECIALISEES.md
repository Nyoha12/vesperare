# 00_INDEX_RECHERCHES_SPECIALISEES

Version : v0.2  
Statut : dossier de stockage documentaire spécialisé. Support d'analyse et d'intégration, pas source de vérité esthétique, pas spécification système.

## Objet

Ce dossier stocke des résultats de recherches externes et des pistes documentaires spécialisées utiles pour Vesperare.

Il sert à rendre les recherches :

```text
faciles à extraire ;
faciles à comparer ;
faciles à relier aux documents internes ;
faciles à intégrer plus tard sans doublonner ;
faciles à déclasser si elles ne sont pas pertinentes ;
utilisables par d'autres conversations ou agents sans créer de doctrine parallèle.
```

Ce dossier ne remplace pas :

```text
docs/05_CARTOGRAPHIE_STYLE_PERSONNEL_TECHNO.md
docs/01_CAHIER_DES_PHENOMENES_SONORES.md
docs/06_CARTOGRAPHIE_INFLUENCES_STYLE_VESPERARE.md
docs/07_PITCH_HARMONIQUES_MICROTONALITE.md
docs/08_PERFORMANCE_SCENES_TRAJECTOIRES_OBJETS.md
docs/12_TECHNO_ORGANIQUE_CADRE_OPERATOIRE.md
docs/13_CONCEPTS_OPERATOIRES_TECHNO_ORGANIQUE_DEPENDANCES.md
docs/14_CONCEPT_CORE_TO_INTERFACE_SYSTEM_REQUIREMENTS.md
```

Il les alimente seulement lorsque les recherches externes permettent de mieux comprendre une notion, une influence ou une méthode.

---

# 1. Principe d'usage

Les recherches externes ne sont pas des preuves directes pour Vesperare.

Elles doivent toujours être traitées comme :

```text
contexte musical ;
contexte historique ;
comparaison méthodologique ;
vocabulaire utile ;
risque à éviter ;
source d'exemples ;
hypothèse à relier aux documents internes.
```

Une recherche externe devient utile uniquement si elle permet de mieux préciser :

```text
une notion musicale ;
un phénomène sonore ;
un rôle d'objet ;
une scène ;
une trajectoire ;
un type de contrôle ;
un garde-fou ;
une sortie ;
une question ciblée à poser à Yohan.
```

---

# 2. Règles anti-doublon

Ne pas intégrer directement une recherche web dans un document source sans vérifier :

```text
quel document interne est la source de vérité ;
si la notion existe déjà ;
si la recherche précise vraiment un manque ;
si elle ne contredit pas le style techno organique ;
si elle ne transforme pas une influence secondaire en centre du projet ;
si elle ne pousse pas vers un prototype ou un mapping prématuré.
```

# 4. Exclusions actuelles

Les recherches sur l'IA d'accompagnement ne sont pas stockées ici pour l'instant.

Décision provisoire :

```text
IA Companion : perspective future ;
mode initial envisagé : écoute / diagnostic / suggestion textuelle ;
accès possible aux données ;
aucun contrôle ;
aucune génération ;
pas dans ce dossier pour l'instant.
```

Un fichier dédié pourra être créé plus tard si cette perspective devient une vraie piste de conception.

---

# 5. Règle d'intégration future

Avant d'intégrer une note de ce dossier dans un document source, appliquer :

```text
1. identifier la notion interne concernée ;
2. vérifier le document source de vérité ;
3. distinguer fait, interprétation, hypothèse ;
4. formuler l'apport en langage Vesperare ;
5. ne pas importer le style ou la méthode en bloc ;
6. n'intégrer que ce qui renforce une décision musicale ou système existante.
```

---

# 6. Statut des notes

Chaque note peut porter un statut :

```text
À EXTRAIRE : source utile mais pas encore synthétisée ;
EXTRAIT : idée résumée mais non intégrée ;
À COMPARER : utile seulement par confrontation avec les docs internes ;
INTÉGRABLE : apport clair vers un document interne ;
À DÉCLASSER : source intéressante mais non pertinente pour Vesperare ;
À RECHERCHER PLUS : piste incomplète.
```
