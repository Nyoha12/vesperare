# 04_PRIORITES_RECHERCHES_ET_ANALYSES_RESTANTES

Version : v0.1  
Statut : plan de recherches et d'analyses restantes. Support documentaire spécialisé, pas source de vérité esthétique, pas spécification système.

## Objet

Ce fichier stocke les recherches et analyses restantes à mener pour Vesperare, dans un format directement exploitable par une autre conversation ou un agent.

Il complète :

```text
00_INDEX_RECHERCHES_SPECIALISEES.md
01_LIVE_ELECTRONICS_NIME_MAPPING.md
02_STYLES_TECHNO_DUBSTEP_TRANCE_ACID_JAZZ_POLYTEXTURE.md
03_SCHEMA_EXTRACTION_INTEGRATION.md
```

Il ne crée pas de nouvelle doctrine. Il sert à organiser les recherches encore utiles, en indiquant :

```text
quand chercher ;
quoi chercher ;
pourquoi chercher ;
quoi extraire ;
quel document interne pourrait être concerné ;
quel risque éviter ;
quelle décision cela pourrait instruire.
```

Les recherches sur l'IA d'accompagnement ne sont pas incluses dans ce fichier v0.1. Elles restent une perspective future séparée.

# 1. Ordre de priorité général

Ne pas lancer toutes les recherches en même temps.

Ordre recommandé :

```text
1. consolider les notions techno-first à partir des documents internes ;
2. identifier les manques réels dans la table NOTION_TO_SYSTEM_SOLIDITY_AUDIT_BATCH_01 ;
3. utiliser les recherches web seulement pour éclairer les manques ;
4. comparer les apports externes aux docs internes ;
5. proposer des intégrations ciblées uniquement dans les documents sources existants.
```

Règle courte :

```text
web après manque identifié,
pas web comme générateur de doctrine.
```
