# 33_FICHE_PERFORMANCE_CORPS_VIVANT_POLYTEXTURE

Version : v0.1  
Statut : fiche de performance jouable. Sans outil, sans patch, sans mapping, sans prototype.

## Objet

Cette fiche extrait une première situation jouable de :

```text
32_SCENES_PIVOTS_DRAMATURGIES_MUSICALES.md
```

Scène-pivot concernée :

```text
SCENE_PIVOT_CORPS_VIVANT_POLYTEXTURE
```

Elle sert à tester musicalement, par le jeu et l’écoute, si la scène tient avant de construire un outil.

Ce document n’est pas :

```text
une partition ;
un morceau ;
un preset ;
un arrangement ;
un patch ;
un prototype ;
une scène Ableton.
```

C’est une fiche de performance ouverte.

---

# 1. Fonction centrale

Question :

```text
comment faire sentir un corps techno vivant,
porté par didgeridoo + guimbardes,
sans réduire ces sources à une couleur,
et sans transformer la polytexture en démonstration ?
```

Fonction centrale :

```text
installer un corps actif, lisible, mouvant,
capable de pression, de densité et d’hypnose,
tout en gardant plusieurs sorties possibles.
```

---

# 2. Sources de départ

## Sources fixes

```text
didgeridoo live ;
guimbardes live.
```

Elles ne sont pas décoratives.

Elles portent la scène.

## Sources possibles mais non obligatoires

```text
sub discret ou synthèse grave minimale ;
tambour sur cadre si présent ;
gong si présent ;
micro-percussions ou objets courts seulement si fonction claire.
```

## Sources à éviter au départ

```text
field recording ;
halo large ;
freeze long ;
loop de remplacement ;
texture d’ambiance ;
polytexture automatisée trop pleine.
```

---

# 3. Configuration minimale jouable

Version la plus nue :

```text
didgeridoo + guimbardes + pulsation implicite.
```

Version avec soutien :

```text
didgeridoo + guimbardes + sub discret.
```

Version avec peau :

```text
didgeridoo + guimbardes + tambour sur cadre.
```

Version avec masse ponctuelle :

```text
didgeridoo + guimbardes + gong ponctuel.
```

Version complète possible :

```text
didgeridoo + guimbardes + sub discret + tambour optionnel + gong optionnel.
```

Règle :

```text
la version complète n’est pas meilleure par défaut.
Elle est seulement plus risquée et plus riche si la hiérarchie reste claire.
```

---

# 4. Durée indicative

La scène peut durer :

```text
3–5 minutes : forme courte, test de prise corporelle ;
6–9 minutes : développement polytextural ;
10–15 minutes : forme longue avec seuils et bifurcations.
```

La durée n’est pas prescrite.

Critère :

```text
la scène dure tant que le corps reste vivant
et que la densité peut encore changer de fonction.
```

---

# 5. Phases de jeu

## Phase 1 — Prise de corps

Objectif : faire exister le corps avant le beat complet.

Gestes possibles :

```text
didgeridoo stable, soufflé, bas, non spectaculaire ;
guimbarde isolée ou en petites réponses ;
sub absent ou très discret ;
aucun effet de remplissage ;
laisser apparaître une pulsation avant de la nommer.
```

Écoute :

```text
est-ce que le corps est déjà là ?
est-ce que l’on sent une pression sans beat explicite ?
est-ce que les sources live sont centrales ?
```

Sorties possibles :

```text
entrer dans une polytexture ;
rester dans une prise nue ;
retirer encore plus ;
faire apparaître un sub discret.
```

## Phase 2 — Articulation vivante

Objectif : faire travailler la polytexture sans la remplir.

Gestes possibles :

```text
guimbardes en alternance de registres ;
micropulses irrégulières mais lisibles ;
didgeridoo qui module les formants ;
sub qui soutient sans couvrir ;
petits accents peau si tambour présent ;
aucune couche permanente inutile.
```

Écoute :

```text
les guimbardes articulent-elles vraiment le rythme ?
le didgeridoo porte-t-il encore la pression ?
la densité reste-t-elle lisible ?
le corps devient-il plus présent ou seulement plus complexe ?
```

Sorties possibles :

```text
augmenter la pression ;
raréfier ;
préparer un seuil ;
basculer vers pré-drop ;
revenir à la prise nue.
```

## Phase 3 — Charge corporelle

Objectif : augmenter l’énergie sans basculer automatiquement vers drop.

Gestes possibles :

```text
sub plus présent mais contrôlé ;
didgeridoo plus pressurisé ;
guimbardes plus insistantes ;
accentuation progressive ;
repeat très court possible sur une cellule, si fonction claire ;
tambour comme renfort, pas comme batterie complète.
```

Écoute :

```text
la charge est-elle physique ?
la pression est-elle tenue ou simplement forte ?
le repeat sert-il l’hypnose ?
les sources live restent-elles devant ?
```

Sorties possibles :

```text
pré-drop ;
retour au corps simple ;
suspension avec trace ;
polytexture plus dense ;
fausse stabilisation.
```

## Phase 4 — Seuil ouvert

Objectif : rendre plusieurs issues possibles.

Gestes possibles :

```text
retirer un appui au lieu d’ajouter ;
laisser une guimbarde devenir signal ;
faire du didgeridoo une tension plutôt qu’un sol ;
préparer un impact peau si tambour présent ;
armer une ponctuation gong si présent ;
laisser un silence court ou un trou de densité.
```

Écoute :

```text
sent-on que quelque chose peut arriver ?
plusieurs issues sont-elles encore crédibles ?
le corps est-il menacé mais pas perdu ?
le seuil est-il musical, pas technique ?
```

Sorties possibles :

```text
résoudre par impact ;
retarder ;
annuler ;
partir vers tension armée ;
partir vers suspension ;
retourner au corps.
```

## Phase 5 — Sortie ou réincorporation

Objectif : sortir sans fermer définitivement.

Gestes possibles :

```text
réduire la polytexture ;
laisser le sub se stabiliser ;
faire revenir didgeridoo comme sol ;
laisser une guimbarde comme trace ;
fermer une queue gong ;
stopper un repeat avant qu’il devienne effet.
```

Écoute :

```text
le retour au corps paraît-il nécessaire ?
la sortie laisse-t-elle une trace ?
la scène peut-elle repartir ailleurs ?
la densité a-t-elle changé de sens ?
```

---

# 6. Gestes interdits ou dangereux

```text
installer une boucle pleine dès le départ ;
faire du didgeridoo une basse automatique ;
faire des guimbardes un décor ethnique ou métallique ;
mettre un sub qui couvre les sources live ;
ouvrir un halo large sans besoin ;
utiliser repeat comme effet ;
faire tourner une cellule jusqu’à remplacement du geste ;
ajouter des couches pour prouver la richesse ;
chercher le drop par réflexe ;
perdre la possibilité de retour au corps.
```

---

# 7. Points de contrôle pendant le jeu

Toutes les 1–2 minutes, vérifier mentalement :

```text
1. Où est le corps ?
2. Qui porte la pression ?
3. Les guimbardes sont-elles audibles comme gestes ?
4. Le didgeridoo garde-t-il son agency ?
5. La densité sert-elle une fonction ?
6. Le sub soutient-il ou masque-t-il ?
7. Une sortie est-elle possible ?
8. Suis-je en train de remplir par peur du vide ?
9. La scène peut-elle bifurquer ?
10. Le live est-il encore la source centrale ?
```

---

# 8. Écoute après performance

Noter rapidement :

```text
durée approximative ;
sources utilisées ;
présence ou absence gong/tambour ;
moment où le corps apparaît ;
moment où la polytexture devient lisible ;
moment où elle devient confuse, si cela arrive ;
moment de plus forte pression ;
seuils réellement perceptibles ;
sorties possibles ;
sortie choisie ;
ce qui a été trop plein ;
ce qui a été trop faible ;
ce qui doit rester ouvert.
```

---

# 9. Critères de réussite

La fiche est réussie si :

```text
le corps est présent avant le beat complet ;
les sources live ne sont pas décoratives ;
la polytexture reste articulée ;
la pression peut augmenter sans écraser ;
le seuil ouvre plusieurs issues ;
la sortie ne ressemble pas à une recette ;
le retour au corps reste toujours possible.
```

---

# 10. Critères d’échec

La fiche échoue si :

```text
la scène devient seulement une texture ;
le didgeridoo devient une basse utilitaire ;
les guimbardes disparaissent ;
le sub prend tout ;
la polytexture est confuse ;
le repeat devient effet ;
le système imaginaire semble jouer à la place du corps ;
le drop devient attendu ;
la sortie est impossible ou arbitraire.
```

# 11. Variantes à tester

Question :

## Variante C — Avec tambour

Question :

```text

```

Question :

```text
le gong ouvre-t-il un seuil sans dissoudre la polytexture ?
```

Question :

