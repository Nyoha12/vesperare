# 36_PROTOCOLE_JOURNAL_REPETITION_ECOUTE_SCENES_PIVOTS

Version : v0.1  
Statut : protocole de répétition et d’écoute. Sans patch, sans mapping, sans prototype, sans asset réel.

## Objet

Ce document donne un protocole pour tester et comparer les trois fiches de performance jouables :

```text
33_FICHE_PERFORMANCE_CORPS_VIVANT_POLYTEXTURE.md
34_FICHE_PERFORMANCE_TENSION_ARMEE_PRE_DROP.md
35_FICHE_PERFORMANCE_SUSPENSION_MATIERE_RETOUR.md
```

Il sert à produire une mémoire d’écoute avant toute décision technique.

Le but n’est pas de valider un système.

Le but est de savoir :

```text
ce qui tient musicalement ;
ce qui s’effondre ;
ce qui appelle un outil ;
ce qui n’a pas besoin d’outil ;
ce qui doit rester live ;
ce qui doit être protégé ;
ce qui peut devenir plus tard une question de prototype ciblée.
```

---

# 1. Principe général

Chaque répétition doit être assez simple pour être rejouée, mais assez ouverte pour rester musicale.

Règle :

```text
jouer peu de choses ;
écouter précisément ;
noter vite ;
comparer plus tard ;
ne pas conclure trop tôt.
```

Le journal ne sert pas à produire des notes parfaites.

Il sert à garder des traces fiables.

---

# 2. Conditions minimales de session

Chaque session doit indiquer :

```text
date ;
durée ;
espace de jeu ;
volume approximatif ;
écoute casque / enceintes / ampli ;
sources utilisées ;
source optionnelle présente ou absente ;
fiches jouées ;
enregistrement oui/non ;
fatigue corporelle ;
état d’écoute.
```

Sources à déclarer :

```text
didgeridoo : présent / absent / lequel ;
guimbardes : nombre approximatif / registres utilisés ;
gong : présent / absent / lequel ;
tambour : présent / absent / lequel ;
sub : absent / discret / actif ;
voix : absente / souffle / fragment / signal ;
field recording : absent / présent / pourquoi ;
loop : absent / trace / shadow / remplacement problématique ;
repeat : absent / articulation / hypnose / effet ;
freeze : absent / bref avec release / problématique.
```

---

# 3. Format court de note après chaque prise

À remplir immédiatement après une prise, en moins de deux minutes.

```text
Fiche jouée :
Durée :
Sources :
Moment où le corps apparaît :
Moment de tension maximale :
Moment de confusion :
Sortie choisie :
Sorties qui restaient possibles :
Ce qui a tenu :
Ce qui a lâché :
Ce qui était trop plein :
Ce qui était trop vide :
Ce qui appelait un outil :
Ce qui doit rester live :
À refaire : oui / non / sous condition.
```

---

# 4. Format long de note après écoute enregistrée

À remplir seulement après réécoute.

## 4.1 Corps

```text
le corps est-il présent ?
à partir de quand ?
par quelle source ?
reste-t-il actif dans les retraits ?
est-il perdu dans les suspensions ?
revient-il naturellement ?
```

## 4.2 Live

```text
didgeridoo agent ou décor ?
guimbardes agents ou décor ?
agency du performeur audible ?
source live masquée ?
loop ou repeat remplace-t-il un geste ?
```

## 4.3 Techno comme fonction

```text
pression présente ?
impact présent ?
hypnose présente ?
tension longue présente ?
répétition active ?
retour au corps possible ?
```

## 4.4 Polytexture / tension / suspension

Selon la fiche :

```text
polytexture lisible ou confuse ?
tension armée ou attendue mécaniquement ?
suspension active ou ambientisée ?
```

## 4.5 Sorties

```text
sortie claire ?
sortie trop brusque ?
sortie trop attendue ?
fausse résolution crédible ?
annulation crédible ?
bifurcation crédible ?
retour au corps crédible ?
```

## 4.6 Risques

```text
ambientisation ;
repeat démonstratif ;
freeze sans release ;
sub trop large ;
source live masquée ;
gong envahissant ;
voix gimmick ;
field recording décoratif ;
perte de corps ;
complexité non lisible.
```

---

# 5. Grille comparative simple

Noter de 0 à 3.

```text
0 = absent / problématique ;
1 = présent mais faible ;
2 = fonctionnel ;
3 = fort / fertile.
```

Critères :

```text
corps ;
pression ;
impact ;
hypnose ;
tension ;
qualité live ;
lisibilité ;
polytexture ;
trace ;
sortie ;
retour au corps ;
risque ambient ;
risque décoratif ;
nécessité d’outil.
```

Attention :

```text
nécessité d’outil élevée n’est pas forcément positif.
Elle indique seulement qu’une fonction musicale demande peut-être un support technique.
```

---

# 6. Protocole par fiche

## 6.1 Corps vivant / polytexture

Tester au moins trois versions :

```text
A. didgeridoo + guimbardes seuls ;
B. didgeridoo + guimbardes + sub discret ;
C. didgeridoo + guimbardes + source optionnelle si disponible.
```

Questions principales :

```text
le corps existe-t-il avant le beat complet ?
la polytexture reste-t-elle lisible ?
les guimbardes portent-elles vraiment l’articulation ?
le didgeridoo reste-t-il agent ?
le sub soutient-il ou masque-t-il ?
la scène peut-elle sortir vers tension ou suspension ?
```

À noter particulièrement :

```text
moment où la polytexture devient fertile ;
moment où elle devient confuse ;
source qui porte le retour au corps ;
ce qui demande vraiment un contrôle futur.
```

## 6.2 Tension armée / pré-drop

Tester au moins trois versions :

```text
A. tension sans résolution ;
B. fausse résolution ;
C. résolution sèche ou retour au corps.
```

Questions principales :

```text
la tension change-t-elle de qualité ?
le drop reste-t-il non automatique ?
peut-on retarder sans affaiblir ?
peut-on annuler sans faire panne ?
le repeat sert-il le seuil ou devient-il effet ?
les sources de résolution sont-elles réellement disponibles ?
```

À noter particulièrement :

```text
moment où la résolution devient attendue ;
meilleur moment de refus ;
meilleur moment de retour au corps ;
source qui porte le seuil.
```

## 6.3 Suspension / matière / retour

Tester au moins trois versions :

```text
A. sans halo ;
B. avec matière exposée ;
C. avec retour au corps lent.
```

Questions principales :

```text
le corps reste-t-il latent ?
la matière a-t-elle une fonction ?
la suspension a-t-elle une direction ?
le freeze, s’il existe, a-t-il une release ?
le halo sert-il la forme ?
la sortie est-elle audible ?
```

À noter particulièrement :

```text
moment où la scène risque de devenir ambient ;
matière la plus fertile ;
trace corporelle la plus forte ;
sortie la plus crédible.
```

---

# 7. Décider après trois sessions, pas après une seule

Ne pas conclure après une prise.

Première décision possible après :

```text
au moins trois prises d’une même fiche ;
ou une session contenant les trois fiches ;
ou trois écoutes différées d’un même enregistrement.
```

Une fonction est considérée solide si :

```text
elle revient dans plusieurs prises ;
elle reste claire avec des sources différentes ;
elle supporte une sortie ;
elle ne dépend pas d’un effet spectaculaire ;
elle peut être décrite simplement après écoute.
```

Une fonction est considérée fragile si :

```text
elle exige toujours plus de couches ;
elle perd le live ;
elle devient décor ;
elle ne sait pas sortir ;
elle semble nécessiter un outil avant même d’exister musicalement.
```

---

# 8. Quand une fonction appelle un outil

Un outil devient légitime seulement si la répétition montre une fonction musicale stable mais difficile à tenir manuellement.

Exemples de demandes légitimes :

```text
besoin de garder une trace courte sans remplacer le live ;
besoin de surveiller une release de freeze ;
besoin de protéger les guimbardes du masquage ;
besoin de stabiliser le sub sans effacer didgeridoo ;
besoin de voir quelle sortie reste possible ;
besoin de rappeler le retour au corps ;
besoin de limiter un halo qui ambientise.
```

Demandes non encore légitimes :

```text
faire un beau freeze ;
faire un repeat impressionnant ;
générer une texture ;
compenser une scène qui ne tient pas ;
remplir le vide ;
produire un style ;
automatiser une forme encore floue.
```

---

# 9. Format de décision après répétition

Après une session, écrire :

```text
À garder :
À refaire :
À retirer :
À protéger :
À ouvrir :
À fermer :
Ce qui appelle peut-être un outil :
Ce qui doit rester strictement live :
Prochaine prise :
```

Exemple de conclusion valide :

```text
À garder : guimbardes espacées comme signal de seuil.
À protéger : didgeridoo contre sub trop large.
À ouvrir : suspension vers pré-drop depuis la matière.
À retirer : field recording non nécessaire.
Outil possible : indicateur de release freeze, pas encore patch.
```
