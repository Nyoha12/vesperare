## 1. Verdict global

La v0.2 est correcte dans son principe :

- elle ne detaille pas les styles entiers ;
- elle retient bien des phenomenes ;
- elle separe style, source, morphologie, rythme, spectre, espace, motif, transgression et generation.

Mais plusieurs caracterisations doivent etre nuancees pour ne pas etre reductrices.

Correction generale :

```text
Ajouter pour chaque phenomene ce qui le rend operant :
cadre + variation ;
source + morphologie ;
cycle + accent + role ;
tension + resolution ;
espace + separation ;
sub + vibration + headroom ;
microtonalite + battement + registre ;
instrument + geste + projection.
```

## 2. Verification par element CE

### CE-01 - Techno

Caracterisation v0.2 :

- cadre recuperable ;
- repetition hypnotique ;
- pression corporelle ;
- retour du corps ;
- tension longue ;
- economie active ;
- detail rythmique.

Verification web :

Confirme.

Les travaux de Butler confirment que le fonctionnement de l'EDM/techno ne se reduit pas a kick + boucle. Les couches de metre, les ambiguites d'accent, les tensions entre niveaux de pulsation et les evenements de retour sont centraux.

Risque de reduction dans v0.2 :

Le mot "repetition" peut etre trop pauvre s'il n'inclut pas :

- repetition avec variation perceptible ;
- ambiguite ou conflit metrique ;
- stabilite du cadre malgre micro-deplacements ;
- attente du retour ;
- difference entre pulse entendu et pulse ressenti.

Correction proposee :

Ajouter aux requis :

```text
ambiguite metrique :
niveaux de pulse simultanes :
micro-evenements qui renouvellent la repetition :
ecart entre cadre entendu et cadre ressenti :
```

Statut :

V0.2 valide, a enrichir.

### CE-02 - Techno pseudo-acoustique / realisme augmente

Caracterisation v0.2 :

- son impossible mais physiquement imaginable ;
- geste corporel perceptible ;
- morphologie credible ;
- precision sans signature informatique ;
- source ambigue.

Verification web :

Confirme par la spectromorphologie de Smalley.

La notion de source-bonding confirme qu'un son peut fonctionner parce que l'oreille lui attribue plus ou moins fortement une cause, un geste ou une energie. La morphologie attaque/corps/queue est centrale.

Risque de reduction dans v0.2 :

Dire seulement "source imaginee" peut etre insuffisant.

Il faut distinguer :

- source evidente ;
- source probable ;
- source ambigue ;
- source impossible mais energetiquement credible ;
- source volontairement cachee.

Correction proposee :

Ajouter :

```text
degre de source-bonding :
type d'energie percue : frappe / souffle / friction / pression / rupture / resonance
distance entre source plausible et son obtenu :
```

Statut :

V0.2 valide, a preciser.

- roles de couches ;
- interlocking ;
- densite lisible ;
- contrepoint rythmico-spectral ;
- harmonie de timbres ;

- des roles differencies ;
- une hierarchie perceptive ;

Interlocking ne suffit pas.

Il faut aussi caracteriser :

- role de cadre ;
- role de surface ;
- role de variation ;
- role de densification ;
- role de rupture ;
- relation entre cycle principal et cycle percu.

Correction proposee :

Ajouter :

```text
cadre cyclique partage :
role de surface / role de fond :
cycle entendu vs cycle corporel :
degre de complexite optimale :
```

Statut :

V0.2 valide, a enrichir.

### CE-04 - Dubstep

Caracterisation v0.2 :

- syncope ;
- attente ;
- trou ;
- retour de masse ;
- sub comme matiere ;
- voix criee comme signal ;
- gong / rugissement ;

Verification web :

Partiellement confirme.

Les sources sur dubstep et sound-system confirment l'importance de l'extreme bass, de la culture de diffusion, de la masse physique et de l'ecoute corporelle. La relation sub/systeme/espace est bien essentielle.

Risque de reduction dans v0.2 :

Le couple "trou -> retour de masse" est correct mais peut devenir trop mecanique.

- densite qui se retire ;
- centre grave qui devient ou redevient lisible ;
- rarete du geste.

La source web confirme mieux sub/sound-system que la totalite des choix internes "voix criee / gong". Ces deux derniers restent valides car ils viennent des sources internes Vesperare, mais doivent etre verifies par ecoute situee.

Correction proposee :

Ajouter :

```text
type d'attente corporelle :
forme de desalignement :
forme de realignement :
degre de satisfaction du retour :

```

V0.2 valide pour sub/attente/retour ; voix/gong a verifier par exemples Vesperare.

Confirme par les travaux sur sound-system et sonic dominance.

"Confort" peut affadir la reference.

Il faut garder :

- confort spectral ;
- puissance immersive ;
- vibration ;
- domination ou enveloppement ;
- seuil entre plaisir physique et fatigue.

Ajouter :

```text
immersion corporelle :
pression vibratoire :

```

Statut :

V0.2 valide, mais "confort" doit etre compris comme confort dans une pression forte, pas douceur.

## 3. Verification des elements transversaux ET

### ET-01 - Pression rythmique

Verification :

Confirmee par Butler et Witek.

Point important :

Le groove fonctionne par equilibre entre predictibilite et surprise. La syncopation moderee peut augmenter plaisir et mouvement corporel ; la complexite excessive peut perdre le corps.

Correction proposee :

Ajouter :

```text
degre de complexite optimale :
predictibilite :
surprise :
corps oriente ou perdu :
```

### ET-02 - Techno suspendue

Verification :

Partiellement confirmee par les sources EDM/trance et par les principes d'attente/retour.

Point important :

La suspension longue reste a verifier surtout par ecoute situee Vesperare. Les sources confirment que retrait, build, breakdown, anticipation et retour fonctionnent musicalement, mais elles ne prouvent pas automatiquement qu'une suspension tres longue restera Vesperare.

Correction proposee :

Ajouter :

```text
preuve de memoire :
indice de retour :
pression restante :
seuil ou ca devient ambient :
```

### ET-03 - Espace / halo

Verification :

Confirmee par auditory scene analysis, sound system studies et spectromorphologie.

Point important :

Espace et halo agissent sur fusion, separation, profondeur, distance, masquage et source-bonding. Le halo n'est pas seulement une queue : il modifie la scene auditive.

Correction proposee :

Ajouter :

```text
effet sur fusion/separation :
effet sur source-bonding :
effet sur distance :
masquage induit :
```

### ET-04 - Melodie possible

Verification :

Confirmee indirectement par les sources acid, voix, jazz et trance.

Point important :

Le motif reconnaissable peut fonctionner s'il est lie a une source, une ligne, une tension, un formant, un accent ou une progression de timbre. Le risque reste le lead banal.

Correction proposee :

Ajouter :

```text
origine perceptive du motif :
fonction du motif dans la forme :
degre de memorabilite voulu :
ce qui empeche le lead banal :
```

### ET-05 - Transgression

Verification :

Confirmee par les sources sur sound-system, roughness/microtonalite, voix, gong et psychoacoustique.

Point important :

La transgression peut venir de masse, rugosite, dissonance, sub, espace, rugissement, voix nue ou cri. Elle ne doit pas etre reduite a volume/saturation.

Correction proposee :

Ajouter :

```text
source psychoacoustique de la transgression :
registre dangereux :
duree tolerable :
sortie perceptive :
```

## 4. Corrections prioritaires a appliquer dans une v0.3

### Priorite 1 - Ajouter les "mecanismes de fonctionnement"

Pour chaque element, ajouter une sous-section :

```text
Ce qui fait fonctionner le phenomene :
```

Exemples :

- techno : cadre + ambiguite metrique + micro-variation + retour ;
- acid : boucle courte + accent + slide + mutation timbrale ;
- didgeridoo : bourdon + tractus vocal + formants mobiles + souffle ;
- gong : attaque + partiels inharmoniques + evolution de queue ;
- sound system : vibration + immersion + profondeur + headroom.

### Priorite 2 - Distinguer validation externe et validation interne

Certains points sont confirmes par sources externes :

- groove/syncopation ;
- source-bonding ;
- formants du didgeridoo ;
- accent/slide TB-303 ;
- inharmonicite du gong ;
- source-filtre vocal.

Certains points restent surtout internes Vesperare :

- gong comme drop dubstep ;
- voix criee comme signal precis ;
- techno suspendue tres longue ;
- acid naturalise par didgeridoo ;
- seuil de cliche trance ;
- seuil de microtonalite "belle" vs theorique.

### Priorite 3 - Ne pas ajouter de nouveaux styles

L'audit ne justifie pas d'ajouter :

- reggae ;
- brostep ;
- jazz modal complet ;
- acid house complet ;
- trance complete ;
- electroacoustique complete ;
- traditions percussives.

Il justifie seulement d'affiner les phenomenes deja retenus.

## 5. Questions restantes apres recherche

Ces questions ne peuvent pas etre tranchees par web seul :

1. A partir de quel seuil un son acid naturalise devient trop 303 pour toi ?
2. Quelle duree de techno suspendue garde encore assez de memoire dans ton ecoute reelle ?
3. Quelle voix est desirable : nue, brute, soufflee, criee, fragmentaire, presque chantable ?
4. Quel type de tension jazz te plait vraiment : voicing doux, frottement, resolution retardee, ponctuation ?
5. Quel niveau de sound-system est recherche : confort profond, immersion massive, quasi-domination sonore ?
6. Quelle part de rugissement de gong reste musicale avant de devenir effet spectaculaire ?
7. Quelle densite de polytexture reste corporelle en contexte club ?

## 6. Decision

La caracterisation v0.2 n'est pas fausse.

Elle est cependant encore trop courte sur certains "mecanismes de fonctionnement".

La prochaine version devrait :

- conserver la restriction aux phenomenes retenus ;
- ajouter une section "ce qui fait fonctionner le phenomene" ;
- separer validation externe et validation Vesperare ;
- transformer les corrections ci-dessus en requis techniques propres a chaque element ;
- ne pas ouvrir de nouvelle encyclopedie stylistique.

## 7. Addendum - Bloc D plausibilite acoustique / PA-X

Recherche ciblee ajoutee pendant les arbitrages du 2026-06-20.

Sources principales :

- Smalley : source-bonding, spectromorphologie, morphologie sonore ;
- Gaver : ecoute d'evenements, attributs de source, causes physiques percues ;
- Bregman : auditory scene analysis, groupement et separation de sources.

Conclusion utile :

```text
La plausibilite acoustique doit etre caracterisee par la perception d'une cause, d'une matiere, d'un geste, d'un espace et d'une scene.
Elle ne depend pas directement du fait que le son soit produit acoustiquement ou numeriquement.
```

Correction a garder :

- un son numerique peut etre acceptable s'il reste cause, matiere, geste, espace ou fonction ;
- un son acoustique peut etre fragile s'il devient decoratif, illustratif ou sans fonction ;
- PA-X doit designer une signature electronique identifiable, pas une technologie de production ;
- les cas limites doivent etre testes en contexte : reverb, freeze, shimmer, sidechain, saturation, granularite, sub, pitch, acid, voix.

## 8. Addendum - Reverbs / espace reactif

Recherche ciblee ajoutee pendant les arbitrages du 2026-06-21.

Sources principales :

- Smith / Physical Audio Signal Processing : reverberation artificielle, reflexions, delays, filtres, diffusion, FDN ;
- FLUX:: SPAT : reverb comme espace virtuel, early/cluster/tail, reverberance, heaviness, liveness, infinite/freeze, air absorption, modal density ;
- Dal Santo et al. : late reverberation, diffusion, energy decay, proprietes statistiques de queue ;
- Sound On Sound : shimmer comme reverb pitch-shiftee, utile pour nommer le risque de signature shimmer.

Conclusion utile :

```text
Une reverb ne doit pas etre caracterisee seulement par taille et duree.
Elle implique au minimum direct/early/cluster/tail, densite, diffusion, absorption frequencielle, decay, poids grave, brillance, profondeur, enveloppement et coherence de scene.
```

Correction a garder :

- l'espace/reverb peut integrer un son dans un monde commun ;
- la reverb peut naturaliser un artefact si elle le transforme en cavite, air, metal, queue, souffle, distance ou matiere ;
- le freeze est une fixation/prolongation de matiere ou d'espace, pas automatiquement ambient ;
- le shimmer est fragile parce qu'il porte une signature reconnaissable de reverb pitch-shiftee ;
- les graves de reverb doivent etre traites comme zone critique, car ils peuvent soutenir la pression ou brouiller le sub ;
- les controles doivent separer au moins integration courte, queues longues, freeze, shimmer, espace reactif, halo lock et protections.

## 9. Addendum - REVQ-1 socle spatial et mesure de salle

Recherche ciblee ajoutee pendant les arbitrages du 2026-06-21.

Sources principales :

- REW : mesure par sweep, reponse impulsionnelle, decay, RT60, analyse de salle avec micro de mesure ;
- Smaart / mesures live : transfer function, impulse response, approche systeme de diffusion ;
- Barron et travaux concert hall : relation entre reflections, spatial impression, distance, clarity et envelopment ;
- Lokki et al. : importance perceptive des reflexions laterales et de l'enveloppement musical.

Conclusion utile :

```text
Un seul micro de mesure, deplace a plusieurs positions, peut fournir une analyse utile de tendances de salle.
Il ne donne pas une modelisation parfaite et simultanee de toute la salle.
```

Implications Vesperare :

- la mesure de salle doit guider les corrections et choix de reverb, pas piloter brutalement le mix ;
- les mesures utiles sont : reponse frequentielle, impulse response, decay/RT par bandes, resonances graves, zones de feedback, lisibilite des attaques ;
- la reverb artificielle doit tenir compte de la salle naturelle ;
- dans une salle deja floue, privilegier proximite, early reflections, queues courtes et controle du grave ;
- dans une salle seche, l'espace commun artificiel peut jouer un role plus important ;
- la reverb peut enrichir la perception par largeur, profondeur, taille apparente, enveloppement, fusion/separation de plans et revelation de partiels ;
- l'enrichissement perceptif doit rester compatible avec pression rythmique, clarte du sub, didgeridoo, voix et gong.

## 10. Addendum - REVQ-2 / REVQ-3 reactivite et grave

Recherche ciblee ajoutee pendant les arbitrages du 2026-06-21.

Sources principales :

- Brandtsegg et al. : effets cross-adaptatifs, ou les parametres d'un traitement peuvent etre controles par l'analyse d'un autre signal ou par les relations entre plusieurs signaux ;
- Shure : feedback et gain acoustique potentiel, utile pour rappeler que reverb, micro, salle et diffusion doivent garder des protections ;
- sources reverb precedentes : direct/early/tail, densite, diffusion, absorption, heaviness, air absorption, decay.

Conclusion utile :

```text
La reactivite de la reverb peut etre pensee comme analyse -> decision -> traitement.
Elle peut reagir a la source traitee, a une autre source, a la densite globale, au sub, a la salle ou a la forme musicale.
```

Implications Vesperare :

- les automatismes pro doivent gerer grave, masquage, attaques, brillance, feedback, densite et salle ;
- la reactivite instrumentale doit rester mode/scene : didgeridoo, gong, voix ;
- le vrai sub doit rester majoritairement controle ;
- la sensation d'espace grave peut venir de bas-mediums, harmoniques, souffle, partiels, queue de gong et traces acoustiques ;
- un bloom grave ou un grave reverbere libre peut exister comme exception decidee, pas comme defaut ;
- les controles live doivent separer grave hors reverb, trace acoustique, corps grave, pression et libre.

## 11. Addendum - REVQ-4 / REVQ-5 freeze et shimmer

Recherche ciblee ajoutee pendant les arbitrages du 2026-06-21.

Sources principales :

- Ableton Spectral Time : freeze spectral/granulaire comme capture et relecture de matiere ;
- Sound On Sound : shimmer comme combinaison de pitch shifting et reverb longue ;
- Valhalla DSP : historique du shimmer, feedback, pitch shifting, grains et artifacts comme texture ;
- Eventide : ShimmerVerb comme reverb avec pitch shifters paralleles et couches brillantes/etherees.

Conclusion utile :

```text
Freeze et shimmer ne sont pas une seule famille "ambient".
Le freeze peut etre capture de matiere, spectral hold, prolongation instrumentale ou suspension.
Le shimmer est fragile parce qu'il combine souvent reverb longue, pitch shifting, feedback et brillance reconnaissable.
```

Implications Vesperare :

- le freeze doit capturer une matiere fonctionnelle : gong, didgeridoo, voix, souffle, friction, artefact naturalise, reverb globale ;
- le freeze grave/sub doit rester exception decidee et protegee ;
- un freeze doit avoir une sortie musicale : fade, rupture, aspiration, contraction, release ou retour ;
- le shimmer ne doit pas etre une couleur par defaut ;
- le shimmer utile doit etre naturalise en partiels, metal, air, cavite, microtension, gong ou souffle ;
- le shimmer octave-up ambient/cinematique reconnaissable est un risque PA-X ;
- un shimmer sombre/downshift ou microtonal peut etre plus compatible avec Vesperare que le shimmer classique.

## 12. Addendum - REVQ-6 naturalisation des artefacts

Recherche ciblee ajoutee pendant les arbitrages du 2026-06-21.

Sources principales :

- Ableton Convolution Reverb : convolution par impulse responses et empreintes d'espaces/objets ;
- Sound On Sound : convolution creative comme outil de transformation par empreinte ;
- Cycling 74 reson~ : resonateur comme filtre centre, utile pour penser cavites/resonances ;
- Curtis Roads / Microsound : granularite, microsound, grains et textures.

Conclusion utile :

```text
Un artefact numerique peut etre transforme en matiere perceptive par empreinte, resonance, grain, cavite, espace, friction ou source live.
Le critere n'est pas la methode technique, mais la sortie perceptive : outil reconnaissable ou matiere fonctionnelle.
```

Implications Vesperare :

- convolution et empreintes peuvent donner lieu, surface, cavite ou matiere ;
- resonateurs/source-filter peuvent donner tube, bouche, gong, membrane, metal ou corps ;
- granularite peut devenir souffle, poussiere, friction, queue ou air si elle ne sonne pas outil ;
- saturation/friction et modulation lente peuvent rendre un son trop propre plus physique ;
- l'artefact peut rester perceptible si sa fonction devient rythme, tension, rupture, preparation de retour ou transgression ;
- l'artefact PA-X involontaire doit etre corrige, naturalise ou bloque ;
- l'artefact PA-X volontaire reste possible comme exception explicite.

## 13. Addendum - REVQ-7 halo lock / priorites live

Recherche ciblee ajoutee pendant les arbitrages du 2026-06-21.

Sources principales :

- Ableton Live Manual : automation, envelopes, re-enable automation et distinction entre automatisation et reprise manuelle ;
- Avid Pro Tools : modes d'automation, touch/latch/write comme reference de comportements de controle ;
- Sheridan : controle supervise, utile pour penser humain, automation et priorites de securite.

Conclusion utile :

```text
Le halo lock doit etre pense comme une logique de priorite et de reprise de main.
Un choix live doit pouvoir bloquer une automation artistique,
mais les protections critiques doivent rester au-dessus.
```

Implications Vesperare :

- Auto / Lock / Force / Cut / Momentary / Safe force est plus robuste qu'un simple on/off ;
- le lock bloque les automations artistiques, pas les protections critiques ;
- les protections doivent preferer filtrer, duck-er, deplacer ou convertir avant de couper un halo voulu ;
- le systeme doit signaler quand il bride un choix live pour cause de securite ou de mix vital ;
- les cibles de halo lock doivent pouvoir etre globales ou specifiques : freeze, shimmer, gong, didgeridoo, voix, artefact, grave.

## 14. Addendum - REVQ-8 espaces longs stables

Recherche ciblee ajoutee pendant les arbitrages du 2026-06-21.

Sources principales :

- La Monte Young / drone et ecoute de longue duree : attention portee aux sons soutenus, rapports, battements, continuite ;
- Brian Eno / ambient : musique pouvant rester presente sans imposer une ecoute evenementielle constante ;
- Smalley / space-form : espace, image acousmatique, morphologie et ecoute des formes sonores dans le temps.

Conclusion utile :

```text
La stabilite longue n'est pas une absence de travail musical.
Elle peut deplacer l'attention vers micro-variations, partiels, battements, espace, corps, seuils perceptifs et memoire.
```

Implications Vesperare :

- un espace long stable peut etre valide s'il est decide et fonctionnel ;
- il n'a pas besoin d'etre lie en permanence a une action visible ou une source claire ;
- le danger principal est la derive automatique : decor, flou, masquage, perte de tension non decidee ;
- un watchdog doit signaler les derives sans couper automatiquement un espace long lock/force ;
- la sortie doit etre pensee : cut, fade, contraction, impact, retour grave, retour pulse, silence ou faux retour ;
- la perte longue du corps club est acceptable si decidee et si un retour ou une forme restent possibles.
