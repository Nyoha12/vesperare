## 5. Regles de non-dispersion

### 5.1. Ne pas creer de nouvelle fiche sans fonction

Un nouveau fichier doit servir au moins une de ces fonctions :

- consolider ;
- verifier ;
- questionner ;
- specifier ;
- archiver.

### 5.2. Ne pas multiplier les versions actives

Une fois une nouvelle version active creee, les versions precedentes correspondantes deviennent archives.

Exception :

On peut les rouvrir pour audit de perte.

### 5.3. Ne pas passer en technique sans chainage

Tout choix technique futur doit pouvoir pointer vers :

```text
BESOIN STYLISTIQUE -> BESOIN DE GENERATION -> ELEMENT CE/ET -> REQUIS TECHNIQUE -> SPECIFICATION OU QUESTION
```

### 5.4. Ne pas importer un style entier

Les styles servent seulement de reservoirs de phenomenes.

Question obligatoire :

```text
Quel phenomene precis est emprunte ?
Qu'est-ce qui le fait fonctionner ?
Quel risque de cliche cree-t-il ?
Quelle fonction Vesperare remplit-il ?
```
