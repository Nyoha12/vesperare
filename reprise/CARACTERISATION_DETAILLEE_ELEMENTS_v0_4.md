## 0. Fonction de cette v0.4

Cette v0.4 remplace v0.3 comme document actif de caracterisation.

Elle garde la correction methodologique :

```text
Ne pas caracteriser les styles entiers.
Caracteriser seulement les phenomenes retenus.
Adapter la grille au type d'element.
Ajouter ce qui fait fonctionner le phenomene.
Separer validation externe et hypothese Vesperare.
Ne pas figer encore les modules Max for Live.
```

Elle ajoute la synchronisation avec :

- espaces longs decides ;
- mondes spatiaux ;
- Halo Auto / Lock / Force / Cut / Momentary / Safe force ;
- grave spatial Auto-Pro / Off / Trace / Corps / Pression / Bloom / Libre ;
- fonction forte ;
- distinction decide / subi ;
- didgeridoo libre et accommodable ;
- gong grave comme halo, bloom, souffle ou base microtonale possible.

## 1. Familles de caracterisation

Ces familles indiquent quel type de precision est necessaire. Elles ne sont pas des modules techniques.

### FC-01 - Reservoir stylistique de phenomenes

Pour techno, dubstep, dub/sound system, jazz, trance, acid.

Caracteriser seulement :

- phenomene emprunte ;
- fonction Vesperare ;
- mecanisme musical ;
- condition d'apparition ;
- condition de retrait ;
- risque de cliche.

Ne pas caracteriser :

- style complet ;
- histoire ;
- sous-genres ;
- instrumentation complete ;
- codes non retenus.

Pour pression rythmique, cadre techno, interlocking, syncope, build, retrait, retour.

Caracteriser :

- cycle ;
- accent directeur ;
- accents secondaires ou fantomes ;
- densite d'attaques ;
- complexite optimale ;
- lisibilite corporelle ;
- condition de retour.

## 2. Elements CE

### CE-01 - Techno

Familles utiles :

- FC-01 ;
- FC-02 ;
- FC-09.

Phenomenes retenus :

- cadre recuperable ;
- repetition hypnotique ;
- pression corporelle ;
- retour du corps ;
- tension longue ;
- economie active ;
- detail rythmique.

Ce qui fait fonctionner le phenomene :

- un cadre stable ou recuperable ;
- des niveaux de pulse simultanes : entendu, ressenti, fantome ;
- une repetition renouvelee par micro-evenements ;
- une ambiguite metrique ou accentuelle controlee ;
- une tension entre stabilite du cadre et variation interne ;
- une attente du retour ;
- un retour qui reoriente le corps.

Validation externe :

Confirmee par les analyses EDM/techno de Butler et par les travaux sur groove/syncopation.

Validation Vesperare :

- techno comme socle principal ;
- pas presence obligatoire en surface ;
- longue suspension possible si memoire, pression ou retour restent actifs ;
- timbres techno standards non centraux.

Requis techniques corriges :

```text
presence techno : absent / latent / suspendu / reincarne / explicite / retour
type de cadre : pulse / cycle / motif / pression / memoire
niveaux de pulse simultanes :
ambiguite metrique :
degre de repetition :
micro-evenements :
degre de variation :
lisibilite corporelle :
condition de suspension :
condition de retour :
support naturalise :
risque timbre techno standard :
```

Ne pas caracteriser :

- histoire de la techno ;
- sous-genres ;
- kick/hat/clap/rumble comme obligation ;
- palette sonore techno complete.

Points a clarifier :

- seuil ou suspension perd la memoire techno ;
- type de retour le plus fort pour Vesperare ;
- dose de cadre explicite acceptable.

### CE-02 - Techno pseudo-acoustique / realisme augmente

Familles utiles :

- FC-03 ;
- FC-04 ;
- FC-09.

Phenomenes retenus :

- son impossible mais physiquement imaginable ;
- geste corporel perceptible ;
- morphologie credible ;
- precision sans signature informatique ;
- source ambigue.

Ce qui fait fonctionner le phenomene :

- l'oreille peut imaginer une cause physique ;
- le son garde une morphologie credible : attaque, corps, queue ;
- l'energie percue reste corporelle : frappe, souffle, friction, pression, rupture, resonance ;
- la source peut etre ambigue sans devenir preset ;
- la sur-realite intensifie la source sans supprimer sa plausibilite ;
- le son remplit une fonction musicale identifiable.

Validation externe :

Confirmee par Smalley : source-bonding, morphologie sonore et spectromorphologie.

Validation Vesperare :

- coeur de l'identite sonore : PC present sans sonner ordinateur ;
- ambiguite acoustique acceptee ;
- electronique reconnaissable a eviter.

Requis techniques corriges :

```text
source imaginee :
degre de source-bonding : evident / probable / ambigu / impossible credible / masque
cause physique possible :
matiere :
geste :
type d'energie percue :
attaque :
corps :
queue :
evolution spectrale :
degre de sur-realite :
niveau PA :
fonction : impact / micro-pulse / halo / tension / masse / signal
distance entre source plausible et son obtenu :
risque preset/electronique :
```

Ne pas caracteriser :

- tous les procedes de synthese ;
- catalogue d'effets ;
- sound design spectaculaire sans fonction ;
- imitation realiste gratuite.

Points a clarifier :

- seuil exact entre PA-2 et PA-X ;
- signatures electroniques absolument exclues ;
- types de sur-realite desirables.

Familles utiles :

- FC-02 ;

Phenomenes retenus :

- roles de couches ;
- interlocking ;
- densite lisible ;
- contrepoint rythmico-spectral ;

Ce qui fait fonctionner le phenomene :

- un equilibre entre predictibilite et surprise ;
- une complexite suffisante pour enrichir, pas assez pour perdre le corps ;

Validation externe :

Confirmee par les analyses d'interlocking et par les travaux sur groove/syncopation.

Validation Vesperare :

- Keita Ogawa comme point de depart phenomenologique ;
- pas imitation, pas folklore, pas virtuosite decorative ;
- polytexture comme facette activable, pas permanente.

Requis techniques corriges :

```text
nombre de couches actives :
role de chaque couche :
couche directrice :
cadre cyclique partage :
cycle principal :
cycles secondaires :
cycle entendu vs cycle corporel :
densite d'attaques :
registre de chaque couche :
occupation medium :
degre d'interlocking :
degre de complexite optimale :
degre de lisibilite corporelle :
condition de retrait :
risque demonstration/folklore :
```

Ne pas caracteriser :

- style complet de Keita Ogawa ;
- traditions percussives ;
- virtuosite ;
- tout l'instrumentarium percussion.

Points a clarifier :

- densite maximale en club ;
- nombre de couches acceptable ;
- quand la polytexture peut devenir frontale.

- syncope ;
- attente ;
- trou ;
- retour de masse ;

Ce qui fait fonctionner le phenomene :

Validation externe :

Validation Vesperare :

- voix criee et gong/rugissement viennent surtout des sources internes ;
- ces elements restent valides comme hypotheses artistiques Vesperare, a clarifier conceptuellement ;
- pas wobble, pas brostep, pas drop automatique.

Requis techniques corriges :

```text
type de syncope : placement / masse / texture / voix / metal
duree du trou :
type d'attente corporelle :
forme de desalignement :
forme de realignement :
source de masse :
stabilite du grave :
rarete du retour :
degre de satisfaction du retour :
relation au didgeridoo :
niveau de transgression :
source interne/externe du rugissement :
risque drop automatique :
risque basse electronique :
```

Ne pas caracteriser :

- wobble bass ;
- brostep ;
- histoire du dubstep ;
- sound design de basse electronique standard.

Points a clarifier :

- frequence acceptable des retours de masse ;
- seuil ou gong/rugissement devient effet spectaculaire ;
- compatibilite sub/didgeridoo.

## 4. Validations externes vs validations Vesperare

### 4.1. Valide par sources externes

- groove, pulse, ambiguite metrique et micro-variation ;
