Cette fiche decrit ce que la generation doit rendre possible musicalement. Elle ne definit pas encore une architecture technique, une interface, des pistes, des devices ou des mappings.

Regle generale :

Un objet genere ne doit jamais etre seulement "un son", "un pattern" ou "une texture". Il doit etre situe : fonction, contexte, inspiration, pression, plausibilite acoustique, risques et garde-fous.

## BG-01 - Generation d'objets situes

Statut : confirme.

Besoin :

Tout objet genere doit etre defini par sa fonction musicale et son contexte.

Fiche minimale :

```text
Nom :
Scene ou contexte :
Inspiration(s) :
Fonction principale :
Fonctions secondaires :
Regime techno :
Type de pression :
Niveau de plausibilite acoustique :
Regime rythmique :
Regime melodique / spectral :
Support sonore :
Source imaginee :
Relation au live :
Relation au didgeridoo :
Parametres de variation :
Conditions d'apparition :
Conditions de disparition :
Risques stylistiques :
Garde-fous :
```

Risques :

- generer des patterns neutres ;
- choisir un son apres coup ;
- accumuler des couches sans fonction ;
- produire un effet seduisant mais hors intention.

Garde-fou :

La fonction doit etre nommee avant les parametres.

## BG-02 - Generation sous plausibilite acoustique

Statut : confirme.

Besoin :

La generation sonore doit produire ou selectionner des supports dont l'origine reste acoustiquement plausible ou ambigue, sans identification electronique evidente.

Descripteurs :

```text
source imaginee :
cause possible :
matiere :
geste :
attaque :
corps :
queue :
espace :
partiels :
formants :
rugosite :
instabilite :
niveau PA :
risque de preset :
```

Niveaux :

- PA-0 : acoustique direct ;
- PA-1 : acoustique etendu ;
- PA-2 : ambigu plausible ;
- PA-3 : hybride fragile ;
- PA-X : electronique reconnaissable a eviter.

Risques :

- son efficace mais electronique reconnaissable ;
- effet numerique qui prend le dessus ;
- field recording decoratif ;
- naturalisation forcee.

Garde-fou :

Un son peut etre impossible, mais il doit laisser imaginer une cause physique.

## BG-03 - Generation de cadres techno sans timbres techno

Statut : confirme.

Besoin :

La generation doit permettre des cadres techno perceptibles sans imposer kick, hat, clap, rumble ou lead electroniques standards.

Objets a produire :

- impact-fondation naturalise ;
- micro-pulse naturalise ;
- cadre revenant ;
- motif repetitif vivant ;
- variation faible mais identifiante ;
- tension longue ;
- retour de cadre ;
- masse grave plausible ;
- techno latente ou suspendue.

Descripteurs :

```text
fonction techno :
support naturalise :
regime de presence techno :
degre de pulse :
degre de repetition :
type de variation :
type de pression :
zone spectrale :
lisibilite corporelle :
condition de retour :
risque de cliche techno :
```

Risques :

- techno standard ;
- boucle morte ;
- naturalisation decorative ;
- perte de la fonction techno.

## BG-05 - Generation de techno suspendue longue

Statut : confirme.

Besoin :

La generation doit permettre des scenes longues sans pulse techno explicite, tout en gardant pression, memoire ou direction.

Types de suspension :

- pulse fantome ;
- suspension spatiale ;
- suspension spectrale ;
- suspension instrumentale ;
- pre-retour ;
- suspension rituelle ;
- faux retour.

Elements generables :

- respiration cyclique ;
- centre grave latent ;
- halo a periodicite faible ;
- motif fragmentaire ;
- micro-variation hypnotique ;
- tension de partiels ;
- espace actif ;
- attente de densification.

Retours possibles :

- retour du grave ;
- apparition progressive d'une pulsation ;
- contraction de l'espace ;
- densification d'attaques ;
- alignement de couches ;
- transformation d'un souffle en pulse ;
- resolution microtonale ;
- impact rare.

Risques :

- ambient decoratif ;
- attente vague ;
- retour arbitraire ;
- absence de forme.

## BG-07 - Generation melodique multi-regime

Statut : confirme.

Besoin :

La generation melodique doit permettre plusieurs regimes, sans imposer une melodie permanente ni bannir le motif reconnaissable.

Regimes :

- melodie absente ;
- centre ressenti ;
- accent-melody ;
- basse-motif ;
- melodie de timbre ;
- melodie de partiels ;
- ligne acid naturalisee ;
- fragment vocal ;
- hook minimal ;
- melodie microtonale.

Conditions d'acceptation :

- source ou origine plausible ;
- fonction de pression ou de forme ;
- relation a un regime rythmique ;
- mutation timbrale possible ;
- compatibilite avec le grave ;
- capacite a disparaitre ou se transformer ;
- absence de couleur stylistique trop identifiable.

Risques :

- hook generique ;
- lead banal ;
- acid line autonome ;
- melodie trop emotionnelle/cliche ;
- vocal hook pop.

Garde-fou :

Un motif melodique doit etre lie a matiere, voix, grave, partiels, accent ou scene.

## BG-15 - Generation par superposition d'inspirations

Statut : confirme.

Besoin :

La generation doit pouvoir combiner plusieurs inspirations par fonctions, sans collage stylistique.

Regle :

Une scene choisit :

- une fonction principale ;
- deux ou trois fonctions secondaires maximum ;
- un regime de pression dominant ;
- un niveau de plausibilite acoustique ;
- un niveau de densite ;
- une relation au live.

Combinaisons utiles :

- techno + Keita ;
- techno + dubstep ;
- techno + trance ;
- techno + acid ;
- techno + jazz ;
- techno + voix ;
- dubstep + microtonalite ;
- trance + voix ;
- acid + didgeridoo ;
- gong + jazz ;
- sound system + techno.

Risques :

- calques de style ;
- elements qui se concurrencent ;
- surcharge ;
- sons electroniques reconnaissables ;
- inspiration visible mais fonction absente.

Garde-fou :

Les inspirations sont reservoirs de phenomenes, pas masques stylistiques.

## BG-17 - Monopoles perceptifs

Statut : confirme.

Besoin :

La generation doit respecter certains monopoles perceptifs pour eviter la confusion.

Monopoles :

- sub : une seule source principale de vrai grave ;
- articulation : une seule couche directrice d'accent-melody ;
- brillant continu : l'aigu ne doit pas devenir tapis permanent ;
- halo long : queues longues limitees a fonctions ou moments ;
- transgression : une crise principale a la fois.

Risques :

- grave instable ;
- plusieurs directions concurrentes ;
- fatigue aigue ;
- flou ;
- empilement de transgressions.

Garde-fou :

Ces monopoles sont des garde-fous perceptifs, pas des interdictions absolues.

## BG-18 - Verification situee des objets generes

Statut : a preparer, non performance live detaillee.

Besoin :

Chaque type d'objet genere doit pouvoir etre verifie par une relation musicale situee.

Format :

```text
Relation a verifier :
Situation :
Signe de reussite :
Signe d'echec :
Signe de caricature :
Ce que cette verification peut corriger :
Ce qu'elle ne permet pas de conclure :
```

Exemple :

```text
Relation :
Didgeridoo comme grave partage.

Situation :
Didgeridoo libre + masse grave rare.

Reussite :
Le didgeridoo reste perceptible comme corps vivant et la masse renforce le sol.

Echec :
Le didgeridoo devient decor ou est masque.

Caricature :
Le systeme protege tellement le didgeridoo que la pression disparait.
```

Garde-fou :

Une verification situee n'est pas un test technique lourd.

## BG-19 - Garde-fous de glissement

Statut : confirme.

Besoin :

La generation doit etre relue contre les glissements critiques.

Glissements a eviter :

- intention -> parametre technique ;
- instrument live -> input audio ;
- verification situee -> protocole lourd ;
- aide systeme -> decision expressive ;
- techno organique -> collection d'effets ;
- complexite -> accumulation ;
- naturalisation -> decor naturel ;
- didgeridoo libre -> didgeridoo ignore ;
- transgression -> agression ;
- microtonalite -> theorie ;
- melodie -> lead banal ;
- espace -> ambient.

Regle finale :

Si un objet genere ne peut pas etre relie a une inspiration situee, une fonction, un phenomene sonore, un risque et un garde-fou, il reste en hypothese.
