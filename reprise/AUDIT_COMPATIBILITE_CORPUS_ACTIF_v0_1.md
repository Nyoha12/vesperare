## 2. Compatibilites confirmees

### 2.1. Techno centrale vs techno suspendue tres longue

Tension apparente :

- Vesperare garde la techno comme socle stylistique principal ;
- la techno peut etre suspendue tres longtemps.

Resolution :

Ces deux exigences sont compatibles.

La techno n'est pas obligee d'etre presente en surface.

Elle peut rester presente par :

- memoire ;
- tension ;
- cadre recuperable ;
- pression latente ;
- retour du corps ;
- attente ;
- espace actif ;
- retrait decide.

Conclusion :

```text
Techno absente en surface != techno absente stylistiquement.
```

Pas de correction requise.

### 2.2. Pression rythmique prioritaire vs retrait long du corps

Tension apparente :

- la pression rythmique est prioritaire ;
- le retrait long du corps doit rester possible.

Resolution :

La priorite rythmique signifie que le corps doit etre pense, pas necessairement active en permanence.

Le retrait long du corps est compatible si :

- il est decide ;
- il a une fonction perceptible ;
- il garde une memoire, une tension ou une consequence ;
- il ne vient pas d'une automation non voulue.

Conclusion :

```text
Perte du corps subie = risque.
Retrait du corps decide = fonction possible.
```

Pas de correction requise.

### 2.3. Plausibilite acoustique vs hors-monde

Tension apparente :

- les sons electroniques reconnaissables doivent etre evites ;
- certains espaces ou sons peuvent etre impossibles, hors-monde ou tres transformes.

Resolution :

Le corpus distingue correctement :

- plausibilite physique ;
- cause perceptive ;
- source-bonding ambigu ;
- impossible credible ;
- hors-monde decide.

Un son peut etre impossible s'il garde une cause, une matiere, un geste, un monde ou une fonction.

Conclusion :

```text
Impossible credible = compatible.
Preset electronique identifiable = risque.
```

Pas de correction requise.

### 2.4. Naturalisation vs decor

Tension apparente :

- naturaliser les sons est un besoin ;
- la naturalisation peut devenir decor.

Resolution :

La distinction est maintenant stable :

- naturalisation comme appartenance, cause, geste, matiere, pression ou monde : besoin ;
- naturalisation comme decoration non decidee : risque ;
- decor decide : possible ;
- decor impose par l'outil : risque.

Conclusion :

```text
Naturalisation fonctionnelle = besoin.
Naturalisation illustrative non decidee = glissement.
```

Pas de correction requise.

### 2.5. Didgeridoo libre vs didgeridoo integre

Tension apparente :

- le didgeridoo doit rester libre ;
- il peut aussi etre traite, adapte, guide harmonique ou source fonctionnalisee.

Resolution :

Les modes sont compatibles :

- libre protege ;
- systeme adapte au didgeridoo ;
- didgeridoo adapte au systeme ;
- guide harmonique ;
- grave partage ;
- harmoniques adaptees ;
- source traitee ;
- perturbateur ;
- hybride.

Le point central est que le didgeridoo ne devient pas role fixe.

Conclusion :

```text
Libre = pas assigne par defaut.
Integre = option choisie, reversible, situee.
```

Pas de correction requise.

### 2.6. Une source de vrai sub vs graves libres

Tension apparente :

- une seule fonction porte le vrai sub lisible par defaut ;
- les graves libres doivent rester possibles.

Resolution :

La hierarchie est coherente :

- par defaut : une fonction porte le vrai sub lisible ;
- exceptions : graves libres, partage, soutien, retrait, bloom, didgeridoo guide ;
- limite : securite critique et lisibilite musicale.

Conclusion :

```text
Sub lisible par defaut.
Graves libres comme choix live ou scene, pas comme defaut automatique.
```

Pas de correction requise.

### 2.7. Gong grave vs precision du sub

Tension apparente :

- le gong grave doit pouvoir creer souffle, bloom, masse, base microtonale ;
- le vrai sub doit rester precis.

Resolution :

La compatibilite est claire :

- le gong peut porter richesse, partiels, battements, bas-mediums, tension et bloom ;
- le vrai sub reste protege ;
- le grave spatial peut etre Off, Trace, Corps, Pression, Bloom ou Libre.

Conclusion :

```text
Gong grave = richesse perceptive possible.
Sub = precision et lisibilite a proteger.
```

Pas de correction requise.

### 2.8. Halo long vs lisibilite

Tension apparente :

- le halo peut etre long, lock, force ou principal ;
- il peut masquer attaques, sub, didgeridoo, voix ou gong.

Resolution :

Les modes et priorites sont compatibles :

- Auto : adaptation ordinaire ;
- Lock : fonction maintenue ;
- Force : choix fort ;
- Cut : retrait volontaire ;
- Momentary : apparition ponctuelle ;
- Safe force : force avec protections.

La protection critique et la protection de mix vitale restent prioritaires.

Conclusion :

```text
Halo long decide = autorise.
Halo qui detruit une fonction vitale = adapte, signale ou coupe selon priorite.
```

Pas de correction requise.

### 2.9. Espace immense / ambient / decor

Tension apparente :

- espace immense, decor, vide et hors-monde sont possibles ;
- ambient subi est un risque.

Resolution :

Les documents distinguent maintenant :

- espace choisi ;
- decor decide ;
- vide decide ;
- hors-monde decide ;
- ambient subi ;
- grande reverb automatique non decidee.

Conclusion :

```text
Le danger n'est pas l'espace long.
Le danger est l'espace non decide qui remplace la forme.
```

Pas de correction requise.

### 2.10. Voix brute vs integration virtualo-acoustique

Tension apparente :

- voix brute, cri, signal et transgression sont recherches ;
- voix exposee sans integration et auto-tune reconnaissable sont refuses.

Resolution :

Le corpus est compatible :

- voix exposee possible ;
- voix brute possible ;
- cri possible ;
- voix presque chantable possible ;
- integration acoustique ou virtualo-acoustique necessaire ;
- pas de chanson par defaut ;
- pas d'auto-tune reconnaissable ;
- pas de gimmick vocal.

Conclusion :

```text
Voix exposee integree = possible.
Voix exposee sans integration = risque.
```

Pas de correction requise.

### 2.11. Acid naturalise vs squelch / 303

Tension apparente :

- acid, ligne vivante, resonance, torsion et glissement sont souhaites ;
- TB-303, squelch reconnaissable et pattern acid cliche sont refuses.

Resolution :

La fonction acid retenue est phenomenologique :

- ligne vivante ;
- resonance controlee ;
- torsion ;
- variation timbrale ;
- glissement de formant ;
- pression de matiere.

Pas de reproduction de style ou de timbre standard.

Conclusion :

```text
Acid comme comportement de ligne = besoin.
Acid comme signature TB-303 = risque.
```

Pas de correction requise.

### 2.12. Transgression vs confort

Tension apparente :

- masse enorme, rugissement, saturation, microtonalite, espace immense et domination sonore sont acceptes ;
- confort, lisibilite et securite restent importants.

Resolution :

La transgression est definie comme regime contextuel :

- preparation ;
- pic ;
- maintien possible ;
- consequence ;
- sortie ;
- confort ou lisibilite minimale ;
- pas accumulation de crises sans fonction.

Conclusion :

```text
Transgression = fonction situee.
Agression gratuite = glissement.
```

Pas de correction requise.

## 3. Tensions restantes non bloquantes

Ces points ne demandent pas de corriger les fiches maintenant.

Ils doivent etre portes dans la specification conceptuelle.

### 3.1. Fonction forte

Statut :

- definie dans la fiche reverb/espace ;
- reutilisee par stylistique, generation et caracterisation.

Risque :

- devenir une etiquette vague si elle n'est pas traduite en decisions de scene ou controles.

Action future :

- creer une table des fonctions fortes possibles ;
- relier chaque fonction forte a controles, protections et consequences.

### 3.2. Modes live

Statut :

- les modes sont presents ;
- ils ne sont pas encore organises en interface conceptuelle.

Modes concernes :

- DID ;
- GRV ;
- HAL ;
- monde spatial ;
- grave spatial ;
- Auto-Pro ;
- scene ;
- choix live explicite.

Action future :

- specifier quels modes sont des choix de scene ;
- quels modes sont sous la main en live ;
- quels modes sont protections par defaut ;
- quels modes doivent etre signales.

### 3.3. Priorites P0-P5 hors reverb

Statut :

- la matrice P0-P5 est claire dans la fiche reverb ;
- les autres fiches y font reference sans la redevelopper.

Risque :

- si la specification future oublie cette matrice, les modes pourraient redevenir contradictoires.

Action future :

- integrer P0-P5 dans la specification conceptuelle comme regle transversale.

### 3.4. Questions de clarifications artistiques

Statut :

- certaines questions restent ouvertes ;
- elles ne bloquent pas la coherence documentaire.

Exemples :

- niveau exact de domination sound-system ;
- seuil belle microtonalite vs theorie ;
- types de voix les plus desirables ;
- fonction exacte du gong grave selon scene ;
- densite maximale de polytexture ;
- degre de melodie reconnaissable selon morceau ;
- niveau de liberte didgeridoo scene par scene.

Action future :

- les traiter comme questions de specification ;
- ne pas rouvrir les fiches actives sauf contradiction nouvelle.
