Projet : Vesperare
Statut : document actif de formalisation fine avant mise a jour des fiches
Date : 2026-06-20

But :

Formaliser chaque element d'inspiration dans son contexte, puis le traduire en besoins concrets pour le style, la generation sonore, la generation rythmique, la generation melodique, la polytexture et les relations aux instruments live.

Ce document ne met pas encore a jour :

## 0. Methode de non-dispersion

### 0.1. Regle

Chaque inspiration doit etre traitee comme :

```text
contexte reel -> phenomene retenu -> fonction Vesperare -> traduction sonore/rythmique/melodique -> parametres -> risques -> garde-fous
```

On ne retient jamais une etiquette de style seule.

On ne demande pas :

```text
est-ce que ca sonne techno / dubstep / jazz / trance / acid ?
```

On demande :

```text
quelle fonction est active ?
par quel support ?
avec quelle pression ?
avec quelle plausibilite acoustique ?
avec quel risque stylistique ?
avec quelle condition d'apparition ou de retrait ?
```

### 0.2. Grille commune pour chaque inspiration

Chaque inspiration est formalisee avec les dimensions suivantes :

1. Contexte d'inspiration.
2. Ce qui plait vraiment.
3. Ce qui n'est pas a copier.
4. Fonctions Vesperare retenues.
5. Traductions sonores concretes.
6. Traductions rythmiques concretes.
7. Traductions melodiques / harmoniques / spectrales.
8. Besoins de generation.
9. Descripteurs a caracteriser.
10. Modes de presence : absent, latent, ponctuel, frontal, structurel.
11. Combinaisons possibles avec les autres inspirations.
12. Conflits probables.
13. Garde-fous.
14. Exemples d'objets generables.
15. Contre-exemples.

### 0.3. Vocabulaire de reference

Regimes deja poses :

- `PA` : plausibilite acoustique, de `PA-0` acoustique direct a `PA-X` electronique reconnaissable a eviter.
- `PR` : pression rythmique.
- `TS` : techno suspendue.
- `DID` : modes didgeridoo.
- `M` : regimes melodiques.
- `T` : niveaux de transgression.

Ces codes sont des etiquettes de specification, pas des modules techniques figes.

Ils ne designent pas encore :

- des pistes ;
- des devices ;
- des boutons ;
- des pages d'interface ;
- des automatisations obligatoires.

Ils servent a verifier les besoins musicaux avant toute architecture.

## 1. Techno

### 1.1. Contexte d'inspiration

La techno est l'inspiration stylistique principale.

Cas reels ou directions deja cites :

- techno fondatrice ;
- Jeff Mills, notamment pour l'hypnose, l'aspiration, la force de cadre ;
- Robert Hood, notamment pour le temps long, la repetition, le detail rythmique ;
- techno minimale / intense / nocturne ;
- club, nuit, pression corporelle, diffusion.

### 1.2. Ce qui plait vraiment

Ce qui est retenu n'est pas seulement un tempo, un kick ou une drum machine.

Fonctions retenues :

- impact corporel ;
- pulsation ;
- fondation ;
- pression grave ;
- occupation corporelle du spectre ;
- repetition ;
- hypnose ;
- tension longue ;
- densite maitrisee ;
- efficacite de diffusion ;
- intensite nocturne ;
- mystere ;
- cadres qui reviennent sans redondance ;
- musicalite dans les variations ;
- construction de tensions rythmiques, spectrales et timbrales ;
- capacite a produire de la forme sur le temps long.

### 1.3. Ce qui n'est pas a copier

Ne pas copier comme identite dominante :

- kick electronique standard ;
- hi-hat electronique standard ;
- clap techno standard ;
- riser EDM ;
- rumble generique ;
- lead synth banal ;
- dark techno cliche ;
- melodic techno cliche ;
- secheresse froide sans vie ;
- automatisme de boucle.

### 1.4. Fonctions Vesperare retenues

| Fonction techno | Traduction Vesperare |
|---|---|
| Kick | impact-fondation, pas forcement kick |
| Hat | micro-pulsation, grain, frottement, insecte, metal, souffle |
| Rumble | masse grave naturalisee, resonance, didgeridoo, gong, espace grave |
| Loop | cadre revenant, cycle, empreinte, repetition vivante |
| Build | densification, tension, attente, contraction ou expansion |
| Drop / retour | reinstallation du corps, retour de fondation, resolution |
| Automation | transformation organique ou gestuelle |
| Club pressure | pression rythmique, grave, spectrale, spatiale |

### 1.5. Traductions sonores concretes

Supports possibles :

- tambour traite pour impact-fondation ;
- gong sculpte pour impact, masse ou halo ;
- didgeridoo pour fondation, rugosite, partiels, souffle, pulse ;
- voix courte pour signal, accent, impact humain ;
- cloches, bols, metaux pour micro-pulsation ou tension spectrale ;
- souffles courts pour hat naturalise ;
- froissements, grains, frictions pour densite d'attaques ;
- oiseaux/insectes si leur precision, periodicite ou brillance sert une fonction ;
- resonances graves pour masse corporelle.

Le support est accepte seulement si la fonction reste active :

- lisibilite temporelle ;
- attaque ou depart perceptible ;
- propulsion ;
- integration au groove ;
- controle spectral ;
- plausibilite acoustique.

### 1.6. Traductions rythmiques concretes

Regimes privilegies :

- `PR-1` pulse explicite ;
- `PR-2` pulse implicite ;
- `PR-3` interlocking ;
- `PR-5` accent fantome ;
- `PR-8` dissonance metrique controlee ;
- `PR-10` retrait rythmique ;
- `PR-11` retour de cadre.

Objets rythmiques a generer :

- impact rare mais structurant ;
- pulse sans kick ;
- grille ressentie mais cachee ;
- micro-variations sur un cadre stable ;
- accent qui reapparait sans redondance ;
- cycles de longueur differente mais corporels ;
- retour de motif apres suspension ;
- densification progressive sans surcharge.

### 1.7. Traductions melodiques / harmoniques / spectrales

La techno peut etre melodiquement pauvre ou riche selon le contexte.

Pour Vesperare :

- la melodie n'est pas obligatoire ;
- un centre spectral peut remplacer une note ;
- une boucle de partiels peut remplacer une ligne ;
- un motif grave peut porter la direction ;
- un halo peut creer une tension harmonique ;
- une micro-variation de timbre peut produire la musicalite.

Regimes compatibles :

- `M-0` melodie absente ;
- `M-1` centre ressenti ;
- `M-2` accent-melody ;
- `M-3` basse-motif ;
- `M-4` melodie de timbre ;
- `M-5` melodie de partiels.

### 1.8. Besoins de generation

Le systeme doit pouvoir generer :

- des cadres techno perceptibles sans sons techno reconnaissables ;
- des motifs repetitifs vivants ;
- des variations faibles mais identifiantes ;
- des tensions longues ;
- des retours de cadre ;
- des impacts naturalises ;
- des micro-pulsations naturalisees ;
- des masses graves plausibles acoustiquement ;
- des scenes ou la techno est absente longtemps mais recuperable.

### 1.9. Parametres

Parametres a decrire pour tout objet techno-naturalise :

```text
fonction techno :
support naturalise :
regime de presence techno :
degre de pulse :
degre de repetition :
type de variation :
type de pression :
niveau PA :
zone spectrale :
lisibilite corporelle :
condition de retour :
risque de cliche techno :
```

### 1.10. Modes de presence

| Mode | Definition |
|---|---|
| Absent | aucune fonction techno active, sauf memoire formelle eventuelle |
| Latent | tension, cadre ou attente techno sans pulse |
| Suspendu | techno retiree longtemps, avec memoire de retour |
| Reincarne | fonctions techno portees par supports acoustiques |
| Explicite | cadre club clairement perceptible |
| Retour | reinstallation de corps apres suspension |

### 1.11. Combinaisons possibles

- Techno + Keita : pulse et polytexture.
- Techno + dubstep : attente, masse, resolution.
- Techno + jazz : tension harmonique comme matiere.
- Techno + trance : build, suspension, peak non brutal.
- Techno + acid : ligne vivante naturalisee.
- Techno + voix : signal, hook minimal, presence.
- Techno + didgeridoo : fondation libre ou anti-fondation.

### 1.12. Conflits probables

- sons trop electroniques reconnaissables ;
- perte du corps en suspension longue ;
- naturalisation decorative ;
- boucle morte ;
- complexite qui efface la lisibilite ;
- espace trop ambient ;
- force club insuffisante.

### 1.13. Garde-fous

- garder au moins une pression active : rythmique, grave, spectrale, spatiale, attente ;
- definir pour chaque section une memoire ou une absence volontaire de techno ;
- verifier qu'un support naturalise remplit encore sa fonction ;
- bannir les sons PA-X comme identite centrale ;
- privilegier qualite sonore et confort dans l'intensite.

### 1.14. Exemples d'objets generables

```text
Objet : impact-fondation de peau et resonance grave
Fonction : kick naturalise / retour de corps
PA : PA-1 ou PA-2
PR : PR-1 ou PR-11
Risque : son de tambour trop tribal ou kick techno trop evident
```

```text
Objet : hats de souffle metallique
Fonction : micro-pulsation
PA : PA-2
PR : PR-7
Risque : hi-hat electronique deguise
```

```text
Objet : cadre fantome
Fonction : techno latente en suspension longue
PA : PA-2
PR : PR-2 / PR-10
Risque : ambient decoratif si aucune pression ne reste active
```

### 1.15. Contre-exemples

- kick 909 + rumble generique comme fond identitaire ;
- loop de hats standard sous didgeridoo ;
- sidechain EDM visible ;
- dark techno cliche avec sons naturalises poses dessus ;
- groove naturalise qui perd toute lisibilite corporelle.

## 2. Techno pseudo-acoustique / realisme augmente

### 2.1. Contexte d'inspiration

Cette inspiration vient d'une appreciation de certains sons techno qui semblent etre entre synthese et instrument reel : trompette impossible, cloche augmentee, batterie irreelle, percussion synthetique mais gestuelle.

Le point important n'est pas le son electronique lui-meme.

Ce qui compte :

- precision synthetique ;
- expressivite acoustique ;
- sensation d'un geste corporel ;
- lois physiques perceptibles ;
- complexite intelligible ;
- sur-realite.

### 2.2. Ce qui plait vraiment

La sensation qu'un son pourrait etre issu d'un corps ou d'un objet, mais dans une version intensifiee, agrandie, plus precise ou plus impossible.

### 2.3. Ce qui n'est pas a copier

- preset pseudo-acoustique ;
- sound design cinematographique plaque ;
- trompette synth cheap ;
- cloche FM reconnaissable ;
- instrument virtuel realiste mais banal ;
- effet spectaculaire sans fonction.

### 2.4. Fonctions Vesperare retenues

- remplacer les supports techno par des supports plausibles ;
- creer une signature acoustique augmentee ;
- rendre le PC present sans qu'il sonne "ordinateur" ;
- transformer le naturel sans illustration ;
- maintenir la puissance club par des objets non-electroniques reconnaissables.

### 2.5. Traductions sonores concretes

Familles :

- metal impossible mais plausible ;
- peau trop profonde mais credible ;
- souffle trop large mais physique ;
- voix trop resonante mais humaine ;
- gong trop controle mais acoustique ;
- insecte ou oiseau devenu micro-pulse ;
- pierre, bois, cavite, eau ou frottement stylise ;
- didgeridoo etendu par resonances non naturelles mais plausibles.

### 2.6. Traductions rythmiques concretes

Le realisme augmente peut porter :

- impact ;
- micro-pulsation ;
- respiration ;
- rebond ;
- tremblement ;
- oscillation ;
- cycle de queue ;
- friction periodique ;
- battement entre partiels.

### 2.7. Traductions melodiques / harmoniques / spectrales

- melodie de formants ;
- melodie de partiels ;
- glissement de resonance ;
- centre spectral instable ;
- battement microtonal ;
- tension entre bruit colore et hauteur ressentie.

### 2.8. Besoins de generation

Le systeme doit pouvoir evaluer ou produire :

- un niveau `PA` ;
- une source imaginee ;
- une cause physique possible ;
- une matiere possible ;
- un geste possible ;
- une fonction musicale ;
- un risque d'identification electronique.

### 2.9. Parametres

```text
source imaginee :
matiere :
geste :
attaque :
queue :
resonance :
instabilite :
complexite spectrale :
degre de sur-realite :
niveau PA :
fonction techno ou non-techno :
```

### 2.10. Modes de presence

- discret : qualite du son seulement ;
- structurel : tout un motif repose sur cette ambiguite ;
- frontal : un son impossible devient evenement ;
- transgressif : rugissement, cri, masse, saturation organique.

### 2.11. Combinaisons possibles

- avec techno : fonction club naturalisee ;
- avec trance : halo ou expansion ;
- avec acid : filtre comme gorge, resonance ou friction ;
- avec jazz : champ harmonique de partiels ;
- avec dubstep : masse grave plausible ;
- avec voix : choeur spectral, cri transforme.

### 2.12. Conflits probables

- devenir sound design abstrait ;
- perdre la fonction musicale ;
- trop ressembler a un preset ;
- trop illustrer la nature ;
- confondre plausibilite acoustique et douceur.

### 2.13. Garde-fous

- chaque son doit avoir une fonction situee ;
- un son PA-X ne doit pas etre central ;
- l'ambiguite est meilleure que l'imitation ;
- la puissance est autorisee si elle reste physiquement credible.

### 2.14. Exemples

```text
Objet : cloche-grain respirante
Fonction : micro-pulsation + halo
Risque : clochette decor
Garde-fou : ancrage PR clair ou tension spectrale forte
```

```text
Objet : souffle-sub
Fonction : pression grave respirante
Risque : bruit de vent ambient
Garde-fou : relation au corps et au retour techno
```

## 3. Keita Ogawa / polytexture percussive

### 3.1. Contexte d'inspiration

Keita Ogawa sert de point de depart pour analyser des phenomenes de polytexture percussive.

Il ne s'agit pas de copier son style, ni d'importer un style folklorique, latino, cubain ou tribal.

Ce qui est retenu :

- plusieurs sonorites de percussions ;
- grooves complexes ;
- assemblage de couches ;
- richesses d'attaques ;
- harmoniques et sonorites qui s'enchevetrent ;
- rapports aux mesures, temps forts, phrases ;
- fonctions rythmiques subtiles ;
- influence de traditions sans import stylistique direct.

### 3.2. Ce qui plait vraiment

La sensation qu'un groove est fait d'un tissu de petites fonctions :

- une couche porte le corps ;
- une autre cree le frottement ;
- une autre donne une couleur ;
- une autre complete l'attaque ;
- une autre ouvre ou ferme le spectre ;
- une autre fait entendre un contrepoint.

### 3.3. Ce qui n'est pas a copier

- virtuosite decorative ;
- solo de percussion ;
- folklore identifiable ;
- demonstration de complexite ;
- surcharge des mediums ;
- perte de l'ancrage techno ;
- groove trop humainise de facon aleatoire.

### 3.4. Fonctions Vesperare retenues

| Fonction | Description |
|---|---|
| Densification du groove | ajouter activite sans changer le cadre |
| Alternative aux hats | micro-pulsations acoustiques |
| Contrepoint techno | couche qui contredit ou enrichit la pulsation |
| Transition | matiere qui prepare changement |
| Texture principale ponctuelle | polytexture devient le centre |
| Finesse rythmique | accents, queues, attaques, placements |
| Harmonie de timbres | couches percussives creent une couleur |
| Tissage rythmico-spectral | entre percussif, harmonique, spectral |

### 3.5. Traductions sonores concretes

Sources compatibles :

- peaux seches ;
- peaux profondes ;
- bois ;
- cloches ;
- petits metaux ;
- frottements ;
- souffles courts ;
- pierres, graines, objets ;
- percussions tonales ;
- voix consonantique ;
- didgeridoo attaque ;
- gong frappe court ;
- queues resonantes coupees.

### 3.6. Traductions rythmiques concretes

Regimes :

- `PR-3` interlocking ;
- `PR-5` accent fantome ;
- `PR-7` densite d'attaques ;
- `PR-8` dissonance metrique ;
- `PR-12` rythme instrumental live.

Structures possibles :

- cycle principal stable + micro-cycle decale ;
- attaques rares + densite de grains ;
- couche grave lente + couche aigue rapide ;
- cellule courte repetee + couche de reponse ;
- contretemps subtil + accent fantome ;
- alternance de mains imaginaires ;
- pattern qui semble complexe mais se ressent simplement.

### 3.7. Traductions melodiques / harmoniques / spectrales

La polytexture peut produire :

- harmonie de timbres ;
- melodie par partiels de percussions ;
- centre spectral par accumulation ;
- microtonalite par cloches/gongs/peaux ;
- variation de couleur sans changement de notes ;
- contrepoint entre registres.

### 3.8. Besoins de generation

Le systeme doit generer des couches avec role distinct, pas seulement des notes aleatoires.

Roles possibles :

```text
ancrage :
contrepoint :
ornement :
micro-pulse :
appel :
reponse :
queue :
halo :
texture de transition :
accent fantome :
```

### 3.9. Parametres

```text
nombre de couches :
role de chaque couche :
registre :
type d'attaque :
longueur de queue :
densite :
cycle :
decalage :
accent principal :
accent secondaire :
degre d'interlocking :
degre de lisibilite corporelle :
occupation medium :
relation a la techno :
relation au live :
```

### 3.10. Modes de presence

- absent : pas de polytexture ;
- discret : micro-pulses seulement ;
- fonctionnel : densifie le groove ;
- transitionnel : prepare une scene ;
- frontal : texture principale ponctuelle ;
- transgressif : densite extreme controlee.

### 3.11. Combinaisons possibles

- avec techno : hats, contrepoints, densification ;
- avec dubstep : syncope polytexturale ;
- avec jazz : finesse de placement et ponctuation ;
- avec trance : snare roll naturalise, build par couches ;
- avec acid : torsion de timbre dans une couche ;
- avec voix : consonnes, souffles, cris courts ;
- avec didgeridoo : reponses interlockees aux attaques.

### 3.12. Conflits probables

- saturation des mediums ;
- trop de couches en meme temps ;
- perte de la pulsation ;
- trop de hasard ;
- trop de virtuosite ;
- style folklorique reconnaissable.

### 3.13. Garde-fous

- limiter le nombre de couches actives selon la scene ;
- donner un role a chaque couche ;
- reserver un espace au grave et au live ;
- maintenir un accent ou une pression corporelle ;
- pouvoir retirer la polytexture d'un geste ;
- eviter les banques de sons trop connotees.

### 3.14. Exemples

```text
Objet : micro-pulse de peaux/bois
Fonction : alternative aux hats
PR : PR-7
PA : PA-1 / PA-2
Risque : percussion ethnique
```

```text
Objet : contrepoint a 5 attaques contre cadre 4/4
Fonction : dissonance metrique corporelle
PR : PR-8
Risque : exercice rythmique abstrait
```

## 4. Dubstep

### 4.1. Contexte d'inspiration

Le dubstep n'est pas importe comme style entier.

Il est retenu pour :

- syncope ;
- attente ;
- drop rare / declenchable ;
- resolution rythmique ;
- sub ;
- masses graves ;
- voix criees ;
- gongs / rugissements ;
- microtonalite grave.

### 4.2. Ce qui plait vraiment

La capacite a mettre le corps en tension par :

- deplacement de masse ;
- absence avant retour ;
- grave comme matiere ;
- drop attendu mais pas banal ;
- resolution qui satisfait sans etre seulement "retomber sur le 1".

### 4.3. Ce qui n'est pas a copier

- wobble bass reconnaissable ;
- brostep agressif ;
- drop automatique ;
- sound design de basse electronique evident ;
- schema build/drop prefabrique ;
- violence sonore gratuite.

### 4.4. Fonctions Vesperare retenues

| Fonction dubstep | Traduction Vesperare |
|---|---|
| Syncope | deplacement de matiere, trou, retard, anticipation |
| Drop | retour rare de masse, fondation ou resolution |
| Sub | centre de gravite spectral |
| Bass design | masse grave texturee PA-2 |
| Vocal shout | voix brute, signal, transgression |
| Tension before drop | pression d'attente |
| Resolution | realignement rythmique/spectral/timbral |

### 4.5. Traductions sonores concretes

- didgeridoo filtre ou amplifie comme masse grave ;
- gong grave sculpte ;
- voix criee transformee ;
- sub stable mais non electronique reconnaissable ;
- sub vivant, respirant, rugissant ;
- basse texturale issue de resonances ;
- friction grave ;
- halo grave ferme par un impact.

### 4.6. Traductions rythmiques concretes

Types de syncopes :

- syncope de placement ;
- syncope de masse ;
- syncope de texture ;
- syncope polytexturale ;
- syncope vocale / criee ;
- syncope metallique / gong ;
- trou de fondation ;
- retour decale.

### 4.7. Traductions melodiques / harmoniques / spectrales

Priorite de la basse :

```text
1. masse
2. texture
3. fonction corporelle
4. tension harmonique / microtonale
5. ligne melodique reconnaissable eventuelle
```

Regimes compatibles :

- `M-1` centre ressenti ;
- `M-3` basse-motif ;
- `M-5` partiels ;
- `M-9` microtonalite.

### 4.8. Besoins de generation

Le systeme doit pouvoir generer :

- attente de drop sans drop automatique ;
- tension grave progressive ;
- trous rythmiques utiles ;
- retours de masse ;
- resolution par realignement ;
- sub/masses qui coexistent avec didgeridoo ;
- basses texturees non-electroniques reconnaissables ;
- syncopes de matiere.

### 4.9. Parametres

```text
degre d'attente :
rarete du drop :
type de drop :
source de masse :
stabilite du sub :
type de syncope :
duree du trou :
forme de resolution :
relation au didgeridoo :
niveau de transgression :
risque wobble/brostep :
```

### 4.10. Modes de presence

- absent ;
- latent : attente ou sub fantome ;
- ponctuel : syncope ou trou ;
- structurel : scene construite vers retour de masse ;
- transgressif : drop rare, cri, gong, rugissement.

### 4.11. Combinaisons possibles

- avec techno : retour de cadre + sub ;
- avec Keita : syncope polytexturale ;
- avec jazz : microtension grave ;
- avec trance : pre-drop long ;
- avec acid : basse vivante, filtre de matiere ;
- avec voix : cri comme signal ;
- avec gong : rugissement drop.

### 4.12. Conflits probables

- sub masque didgeridoo ;
- drop trop automatique ;
- basse trop electronique ;
- tension trop spectaculaire ;
- perte de subtilite ;
- violence sonore.

### 4.13. Garde-fous

- drop rare, prepare, jamais automatique par defaut ;
- masse grave controlee par contexte ;
- resolution possible sans explosion ;
- bassline reconnaissable seulement si justifiee ;
- relation au didgeridoo toujours verifiee.

### 4.14. Exemples

```text
Objet : syncope de gong grave
Fonction : deplacement de masse
PR : PR-4
T : T-1 / T-2
Risque : effet trailer
```

```text
Objet : pre-drop sans drop immediat
Fonction : attente longue
TS : TS-5
Risque : frustration sans forme
```

## 5. Dub / sound system

### 5.1. Contexte d'inspiration

Le dub / sound system est lie au dubstep mais doit etre traite separement.

Ce qui interesse :

- qualite de production ;
- profondeur ;
- confort d'ecoute ;
- sound system ;
- flow ;
- espace ;
- science du sub et de l'espace.

### 5.2. Ce qui plait vraiment

L'idee qu'un son peut etre immense, profond, physique et agreable, sans devenir agressif.

### 5.3. Ce qui n'est pas a copier

- reggae ;
- skank ;
- delay dub cliche ;
- tape echo comme signature de style ;
- basse dub typique ;
- espace decoratif.

### 5.4. Fonctions Vesperare retenues

- profondeur active ;
- confort dans l'intensite ;
- relation sub/espace ;
- vide comme pression ;
- separation claire des plans ;
- flow de scene.

### 5.5. Traductions sonores concretes

- reverb physique profonde ;
- delay comme distance ou cavite ;
- sub respirant ;
- espace qui repond aux impacts ;
- halo grave ;
- silence ou trou comme tension ;
- queue de gong ou voix placee en profondeur.

### 5.6. Traductions rythmiques concretes

- rythme spatial ;
- echo comme pulsation ;
- retour de queue sur contretemps ;
- respiration de sub ;
- alternance plein/vide ;
- delay court comme rebond corporel.

## 8. Acid

### 8.1. Contexte d'inspiration

L'acid est retenu comme source de phenomenes, pas comme esthetique a importer.

- ligne vivante ;
- resonance controlee ;
- torsion ;
- hypnose par variation de timbre.

Ce qui est accepte mais dose :

- acidite ;
- accent ;
- matiere liquide / nerveuse.

### 8.2. Ce qui plait vraiment

La sensation d'une ligne qui vit par son timbre autant que par ses notes.

La ligne n'est pas seulement melodique :

- elle se tord ;
- elle parle ;
- elle ouvre/ferme ;
- elle devient pression ;

- module acid autonome ;

## 15. Dimensions transversales a ne pas perdre

Ces dimensions ne sont pas des styles separes, mais elles traversent toutes les inspirations.

Elles doivent etre verifiees pour chaque objet genere, chaque scene et chaque combinaison.

## 15.1. Contour sculpte / nettete sensible

### 15.1.1. Contexte

La "secheresse" techno a ete nuancee.

Ce qui est recherche n'est pas une froideur aride, mais une precision dessinee :

- attaques lisibles ;
- contours nets ;
- durees justes ;
- queues maitrisees ;
- silence utile ;
- halo controle ;
- presence sans bavure ;
- intensite avec gout.

### 15.1.2. Fonction Vesperare

Le contour sculpte sert a :

- rendre la pression rythmique lisible ;
- eviter que les halos effacent les attaques ;
- garder la puissance sans brouiller le spectre ;
- permettre la complexite polytexturale sans confusion ;
- faire sentir qu'un son est exactement a sa place ;
- eviter la boucle molle ou la masse informe.

### 15.1.3. Traductions concretes

Un objet est "sculpte" si ses dimensions sont definies :

```text
attaque : nette / molle / frottee / aspiree / retardee
corps : court / dense / creux / profond / rugueux
queue : absente / courte / moyenne / longue / coupee / resonante
halo : nul / leger / fonctionnel / dominant / dangereux
silence autour : aucun / peu / necessaire / structurel
place spectrale : grave / bas-medium / medium / aigu / halo
role : impact / pulse / signal / texture / tension / resolution
```

### 15.1.4. Besoins de generation

Le systeme doit pouvoir produire des objets avec :

- duree controlee ;
- queue dosable ;
- attaque differenciee ;
- halo separable du corps ;
- densite ajustee ;
- role clair dans la scene ;
- retrait possible quand le contour devient flou.

### 15.1.5. Garde-fous

- si la densite augmente, le halo doit pouvoir baisser ;
- si l'attaque est centrale, le halo ne doit pas la masquer ;
- si le son est long, sa fonction doit etre halo, masse, tension ou espace, pas impact ;
- si le son est court, il peut recevoir un halo selectif pour ne pas devenir sec/aride ;
- une intensite forte ne justifie pas une durete mal dessinee.

### 15.1.6. Contre-exemples

- percussion tres reverberee qui pretend etre un kick ;
- gong long utilise comme impact regulier sans controle ;
- texture riche mais sans bord ;
- son sec au point de devenir froid, plat ou hostile ;
- halo automatique sur tout.

## 15.2. Intensite phenomenale sans agression

### 15.2.1. Contexte

Le projet veut conserver une intensite enorme, parfois proche d'une energie hardcore, mais sans fonder cette intensite sur l'agression sonore gratuite.

L'intensite doit venir de phenomenes musicaux et perceptifs, pas seulement du volume.

### 15.2.2. Sources d'intensite acceptees

L'intensite peut venir de :

- pression rythmique ;
- pression grave ;
- pression spectrale ;
- densite maitrisee ;
- espace immense ;
- rarete d'un evenement ;
- retour de cadre ;
- resolution ;
- rugissement ;
- cri ;
- saturation organique ;
- microtonalite exposee ;
- attente longue ;
- contraste entre nudite et masse.

### 15.2.3. Intensite interdite ou fragile

A eviter comme source principale :

- volume augmente sans travail de matiere ;
- aigus punitifs ;
- saturation dure sans fonction ;
- sub boueux ;
- accumulation de couches sans hierarchie ;
- transgression permanente ;
- agressivite qui remplace la musicalite.

### 15.2.4. Besoins de generation

Le systeme doit pouvoir distinguer :

```text
intensite corporelle :
intensite spectrale :
intensite spatiale :
intensite rythmique :
intensite harmonique :
intensite de rarete :
intensite de transgression :
fatigue potentielle :
confort relatif :
```

Il doit aussi pouvoir produire des intensites sans ajouter automatiquement :

- plus de notes ;
- plus de volume ;
- plus d'aigu ;
- plus de saturation ;
- plus de halo.

### 15.2.5. Garde-fous

- toujours identifier la source de l'intensite ;
- une intensite forte doit avoir une qualite sonore ;
- prevoir une sortie ou transformation ;
- garder le confort comme contrainte musicale, pas comme affadissement ;
- verifier la fatigue spectrale ;
- distinguer peak prepare et agression arbitraire.

## 15.3. Espace / halo / pression spectrale

### 15.3.1. Contexte

L'espace est une pression active, pas un decor.

Le halo est ce qui entoure ou suit l'attaque. Il peut lier, prolonger, ouvrir, transformer ou brouiller.

La pression spectrale est une occupation corporelle et mentale du spectre.

### 15.3.2. Fonctions possibles de l'espace

- profondeur ;
- distance ;
- proximite ;
- scene ;
- vertige ;
- intensite ;
- separation des plans ;
- attente ;
- suspension ;
- retour ;
- espace techno non ambient ;
- confort sound system.

### 15.3.3. Fonctions possibles du halo

- prolonger un impact ;
- lier des evenements ;
- creer une memoire ;
- ouvrir l'espace ;
- soutenir une suspension ;
- porter des partiels ;
- creer une tension harmonique ;
- produire une pression douce ;
- transformer une attaque en champ.

### 15.3.4. Risques du halo

- effacer les attaques ;
- rendre tout ambient ;
- envahir le grave ;
- masquer didgeridoo, voix ou percussion ;
- transformer les objets en nappe ;
- flouter le rythme ;
- devenir un return libre sans decision musicale.

### 15.3.5. Besoins de generation

Le systeme doit pouvoir definir :

```text
qui a droit au halo :
combien de halo :
type de halo : tonal / inharmonique / bruite / spatial / rythmique
zone spectrale du halo :
duree :
relation a l'attaque :
relation au grave :
relation au live :
moment ou le halo doit baisser :
moment ou le halo devient fonction principale :
```

### 15.3.6. Regles concretes

- si la densite d'evenements monte, le halo doit pouvoir baisser ;
- si le live a beaucoup de sustain, le halo generatif doit pouvoir baisser ;
- si le grave est charge, le halo doit eviter le bas ou devenir tres controle ;
- si une attaque porte la fonction principale, le halo doit etre ducke, coupe ou deplace ;
- si la scene est suspendue, le halo peut devenir structurel mais doit garder pression, memoire ou direction ;
- si le halo est tonal, il doit etre relie au regime harmonique/microtonal ;
- si le halo est bruite, il doit etre relie a matiere, espace ou tension.

### 15.3.7. Combinaisons par inspiration

| Inspiration | Usage de l'espace / halo |
|---|---|
| Techno | profondeur nocturne, retour de cadre, espace non decoratif |
| Keita | separation des couches, queues courtes, halo selectif |
| Dubstep | trou, attente, sub/espace, retour de masse |
| Sound system | confort, profondeur, pression par vide |
| Jazz | champ de partiels, tension harmonique |
| Trance | suspension, expansion, peak non brutal |
| Acid | reverb filtree qui parle, resonance qui se tord |
| Voix | presence, choeur spectral, souffle place |
| Microtonalite | battements, halos accordes/desaccordes |
| Didgeridoo | halo adapte au bourdon et aux formants |
| Gong | queue, rugissement, champ inharmonique |
| Tambour | halo court pour sculpter sans flouter |

### 15.3.8. Contre-exemples

- meme reverb large partout ;
- halo permanent dans le bas ;
- espace qui rend le groove illisible ;
- pad ambient sans pression ;
- queue de gong qui remplace toute composition ;
- effet spatial spectaculaire sans fonction.

## 15.4. Minimalisme de concentration / beaute / mystere nocturne

### 15.4.1. Contexte

Le minimalisme n'est pas retenu comme pauvrete, rigidite ou reduction obligatoire.

Il est retenu comme concentration :

- peu d'elements si chaque element est fort ;
- precision des places ;
- repetition non redondante ;
- economie lisible ;
- densite possible mais motivee ;
- attention au contour et a la fonction.

La beaute peut etre fonctionnelle ou plus autonome.

Les etats nocturnes, le mystere, l'incertitude et les transports/enivrements au sens large font partie de l'imaginaire techno de Vesperare, sans devenir une illustration de club ou de substances.

### 15.4.2. Beaute fonctionnelle

Un son peut etre beau parce qu'il remplit exactement sa fonction :

- bonne intensite ;
- bon contour ;
- bonne duree ;
- bonne place spectrale ;
- bonne relation au groove ;
- bonne relation au silence ;
- bonne relation a l'espace ;
- bonne apparition et bonne disparition.

Cette beaute doit etre centrale pour :

- techno ;
- polytexture ;
- impacts ;
- micro-pulsations ;
- retours de cadre ;
- pressions rythmiques.

### 15.4.3. Beaute autonome

Le projet doit aussi permettre des moments ou un son, un halo, une voix, un gong, un champ de partiels ou une suspension valent presque pour eux-memes.

Mais "autonome" ne veut pas dire gratuit.

La beaute autonome doit garder au moins un lien avec :

- une pression ;
- une attente ;
- une suspension ;
- une transition ;
- une exposition ;
- une transgression ;
- une memoire de retour ;
- une presence live ;
- une direction formelle.

### 15.4.4. Mystere nocturne / transport

Cette dimension peut se manifester par :

- froid nocturne non aride ;
- incertitude ;
- tension longue ;
- espace profond ;
- repetition hypnotique ;
- intensite sans explication ;
- clarte partielle ;
- presence humaine fragmentaire ;
- retour de cadre qui aspire ;
- sentiment d'etre porte ou attire.

### 15.4.5. Besoins de generation

Le systeme doit pouvoir produire des objets et scenes avec :


### 15.4.6. Garde-fous

- minimalisme ne doit pas devenir pauvrete sonore ;
- beaute autonome ne doit pas devenir decor ;
- mystere ne doit pas devenir flou ;
- nocturne ne doit pas devenir dark techno cliche ;
- transport ne doit pas devenir euphorie generique ;

### 15.5.7. Garde-fous

- ne pas tout transformer en objets isoles ;
- ne pas tout dissoudre en texture ;
- garder une hierarchie perceptive ;
- eviter que la densite devienne illisible ;
- eviter que le fond mange l'avant-plan ;
- eviter que l'objet principal perde son contour.

## 15.6. Monopoles perceptifs / roles dominants

### 15.6.1. Contexte

Certaines fonctions ne peuvent pas etre portees par trop de sources simultanement sans perdre leur force.

Ces "monopoles" ne sont pas des interdictions absolues. Ce sont des garde-fous perceptifs.

### 15.6.2. Monopole du sub

Regle :

Une seule source doit porter le vrai sub a un moment donne.

But :

- stabilite ;
- headroom ;
- lisibilite ;
- ancrage corporel ;
- coexistence avec didgeridoo et graves acoustiques.

Risques :

- didgeridoo + sub + gong grave en meme temps ;
- grave instable ;
- boue ;
- perte du centre corporel ;
- masquage des partiels utiles.

Garde-fous :

- alternance ;
- grave partage ;
- basse adaptee ;
- sub rare ;
- didgeridoo guide harmonique ;
- sub sous le didgeridoo seulement si musicalement et spatialement clair.

### 15.6.3. Monopole de l'articulation

Regle :

Une seule couche principale doit porter l'accent-melody ou l'articulation directrice.

But :

- lisibilite ;
- hierarchie ;
- direction perceptive ;
- danse possible ;
- tension lisible.

Risques :

- plusieurs accent-melodies concurrentes ;
- polytexture sans plan principal ;
- voix, percussion et acid line qui parlent toutes en meme temps ;
- groove riche mais impossible a suivre.

Garde-fous :

- definir la couche directrice ;
- autres couches en soutien, contrepoint ou texture ;
- changer de couche directrice seulement comme evenement formel ;
- garder un signe clair de retour ou de resolution.

### 15.6.4. Monopole du brillant continu

Regle :

L'aigu ne doit pas devenir un tapis permanent.

But :

- eviter la fatigue ;
- preserver la finesse ;
- garder des evenements lumineux ;
- laisser les attaques respirer ;
- eviter le cliche trance/brillance/EDM.

Risques :

- shimmer constant ;
- cloches aigues permanentes ;
- insectes ou grains trop continus ;
- bruit brillant qui masque les details ;
- peak permanent.

Garde-fous :

- brillance par evenements ;
- aigu en rarete ;
- alternance sombre/lumineux ;
- filtrage selon densite ;
- separation entre halo aigu et micro-pulse aigu.

### 15.6.5. Monopole du halo long

Regle :

Les queues longues doivent etre limitees a certaines fonctions ou moments.

But :

- eviter le flou ;
- proteger les attaques ;
- conserver le groove ;
- eviter la nappe permanente ;
- preserver les moments de vraie suspension.

Risques :

- tout devient ambient ;
- gong long partout ;
- reverb large permanente ;
- voix etiree qui masque ;
- halo qui remplace la composition.

Garde-fous :

- halo long seulement si fonction : suspension, transition, espace, beaute autonome, peak, memoire ;
- ducking ou coupe sur attaques principales ;
- reduction automatique quand densite monte ;
- pas de halo long dans le grave sans raison forte.

### 15.6.6. Monopole de la transgression

Regle :

Une transgression forte doit dominer clairement ou rester absente.

But :

- eviter l'empilement de crises ;
- garder la force du pic ;
- maintenir une sortie possible.

Risques :

- cri + saturation + sub + gong + microtonalite extreme simultanes sans hierarchie ;
- fatigue ;
- effet de demonstration ;
- absence de contraste.

Garde-fous :

- choisir la transgression principale ;
- les autres dimensions soutiennent ou se retirent ;
- preparer et sortir ;
- garder une trace du corps ou du silence.

## 15.7. Registre des glissements a eviter

### 15.7.1. Contexte

Le risque principal n'est pas seulement d'oublier des idees.

Le risque principal est de mal traduire les bonnes idees.

### 15.7.2. Glissements critiques

| Code | Glissement | Exemple | Correction |
|---|---|---|---|
| GL-001 | intention esthetique -> parametre technique | hypnose active -> randomisation | decrire la fonction musicale avant le parametre |
| GL-002 | instrument live -> input audio | didgeridoo -> source sub | garder l'instrument comme presence expressive |
| GL-003 | verification situee -> protocole lourd | tester tout abstraitement | verifier une relation musicale precise |
| GL-004 | aide systeme -> decision expressive a la place de Yohan | auto-arrangeur qui choisit la forme | systeme propose/prepare, Yohan decide |
| GL-005 | techno organique -> collection d'effets | techno + dub + ambient + reverb | traduire fonctions, pas empiler styles |
| GL-006 | complexite -> accumulation | plus de couches = plus riche | role distinct pour chaque couche |
| GL-007 | naturalisation -> decor naturel | field recording sur techno | support doit porter une fonction |
| GL-008 | didgeridoo libre -> didgeridoo non gere | tout le laisse jouer mais tout le masque | liberte + modes d'accommodation |
| GL-009 | transgression -> agression | pic = volume/aigu/saturation | pic = fonction, preparation, sortie |
| GL-010 | microtonalite -> theorie | gamme speciale permanente | battement/tension/resolution situee |
| GL-011 | melodie possible -> lead banal | hook generique | motif lie a matiere, voix, grave ou partiels |
| GL-012 | espace -> ambient | grande reverb partout | espace actif, pression, profondeur, direction |

### 15.7.3. Usage

Chaque besoin futur doit pouvoir etre relu avec ces questions :

```text
Est-ce une fonction musicale ou juste un parametre ?
Est-ce une inspiration ou une copie de style ?
Est-ce que le live garde son autonomie ?
Est-ce que la complexite reste lisible ?
Est-ce que le systeme aide sans decider a la place ?
Est-ce que l'effet est necessaire ?
Est-ce que la scene garde une sortie ?
```

### 15.7.4. Garde-fou de methode

Si un besoin ne peut pas etre relie a :

- une inspiration situee ;
- une fonction ;
- un phenomene sonore ;
- un risque ;
- un garde-fou ;

alors il doit rester en hypothese, pas passer en fiche finale.

## 16. Superpositions entre inspirations

### 16.1. Principe

Toutes les inspirations peuvent etre absentes ou superposees.

La question implicite/perceptible n'est pas centrale.

La question correcte :

```text
quels phenomenes sont actifs ensemble ?
est-ce qu'ils produisent une fonction coherente ?
quel conflit creent-ils ?
quel garde-fou les tient ?
```

### 16.2. Combinaisons a fort potentiel

| Combinaison | Resultat possible | Risque |
|---|---|---|
| Techno + Keita | techno polytexturale, hats naturalises, contrepoint | surcharge / demo |
| Techno + dubstep | pression grave, attente, retour de masse | drop cliche |
| Techno + trance | long build, suspension, retour de cadre | euphorie prefabriquee |
| Techno + acid | boucle vivante, timbre qui module | son acid reconnaissable |
| Techno + jazz | tension longue harmonique | perte de corps |
| Techno + voix | signal humain, hook minimal | chanson |
| Keita + jazz | finesse rythmique + ponctuation | intellectualisation |
| Keita + trance | build par couches | roll EDM naturalise mal |
| Dubstep + microtonalite | grave instable, tension de sub | boue grave |
| Dubstep + voix | cri/drop/signal | agressivite |
| Trance + voix | suspension vocale, choeur spectral | vocal trance cliche |
| Acid + didgeridoo | formants, tube, ligne vivante | gadget |
| Gong + jazz | champ de partiels | new age |
| Gong + dubstep | rugissement/drop | trailer |
| Sound system + techno | puissance confortable | dub/reggae |

### 16.3. Regle de surcharge

Une scene ne doit pas additionner les inspirations comme des calques de style.

Elle doit choisir :

- une fonction principale ;
- deux ou trois fonctions secondaires maximum ;
- un regime de pression dominant ;
- un niveau de plausibilite acoustique ;
- un niveau de densite ;
- une relation au live.

### 16.4. Exemple de scene coherente

```text
Scene : techno suspendue de grave vocal/didgeridoo
Inspiration principale : techno suspendue + sound system
Inspiration secondaire : dubstep pour attente, jazz pour champ de tension
Fonction principale : pression d'attente
Support : didgeridoo libre, sub respirant, voix souffle, halo de gong
PR : PR-2 / PR-9 / PR-10
M : M-1 / M-5 / M-9
PA : PA-2
T : T-1 avec possibilite T-3
Risque : ambient decoratif ou boue grave
Garde-fou : retour possible par cadre rythmique ou resolution de grave
```

### 16.5. Exemple de scene incoherente

```text
Scene : kick techno + wobble bass + congas + supersaw + chant
Probleme :
- supports electroniques reconnaissables ;
- trop de styles simultanes ;
- aucune traduction acoustique ;
- collage ;
- fonctions concurrentes non hierarchisees.
```

## 18. Banque d'exemples sonores situes v0.1

Ces exemples ne sont pas des prescriptions de composition.

Ils servent a verifier que chaque inspiration peut devenir un objet musical concret, avec fonction, risque et garde-fou.

Chaque exemple doit etre lu comme :

- un cas de verification ;
- un exemple imaginaire ;
- un support d'analyse ;
- un contrepoint aux caricatures possibles.

### 18.1. Techno reincarnee : pulse de peau sans kick

```text
Contexte :
Techno perceptible mais sans kick electronique.

Objet :
Impact sourd de peau profonde, coupe court, double par une resonance grave tres controlee.

Fonction :
Impact-fondation.

Parametres critiques :
attaque nette, queue courte, grave stable, repetition lisible, peu de halo.

Risque :
Tambour tribal ou kick techno deguise.

Garde-fou :
Garder la fonction club mais eviter le pattern percussif style-identifiable.
```

### 18.2. Techno suspendue : boucle fantome de souffle

```text
Contexte :
Longue section sans pulse explicite.

Objet :
Souffle cyclique tres bas dans la texture, presque confondu avec l'espace.

Fonction :
Memoire techno latente et respiration.

Parametres critiques :
periodicite faible, presence subtile, pas de beat frontal, tension maintenue.

Risque :
Ambient decoratif.

Garde-fou :
Conserver une attente de retour ou une pression d'espace.
```

### 18.3. Techno pseudo-acoustique : metal impossible mais plausible

```text
Contexte :
Besoin d'un son precis et presque synthetique sans identification electronique.

Objet :
Cloche metallique courte avec partiels trop nets pour etre realistes, mais comportement de frappe credible.

Fonction :
Accent lumineux / micro-signal.

Parametres critiques :
attaque physique, partiels stables, queue controlee, brillance rare.

Risque :
Preset FM ou cloche trance.

Garde-fou :
Faire entendre une cause possible : metal frappe, cavite, objet.
```

### 18.4. Keita / polytexture : tissage de peaux, bois et souffle

```text
Contexte :
Densifier le groove sans ajouter un hat standard.

Objet :
Trois couches : bois court en contretemps, peau tres douce sur cycle decale, souffle consonantique comme grain.

Fonction :
Micro-pulsation + contrepoint + densite articulee.

Parametres critiques :
role de chaque couche, registre separe, accents non concurrents, medium controle.

Risque :
Percussion world / demonstration.

Garde-fou :
Une couche porte l'articulation, les autres soutiennent.
```

### 18.5. Dubstep : trou de masse avant retour

```text
Contexte :
Attente de drop sans schema dubstep automatique.

Objet :
Retrait soudain du grave, maintien d'un halo serre, puis retour de masse par gong grave + sub respirant.

Fonction :
Pression d'attente et resolution corporelle.

Parametres critiques :
duree du trou, rarete du retour, stabilite du grave, sortie claire.

Risque :
Drop cliche ou effet trailer.

Garde-fou :
Resolution possible par masse, rythme, spectre ou microtonalite, pas seulement explosion.
```

### 18.6. Sound system : vide profond et sub confortable

```text
Contexte :
Intensite forte mais agreable.

Objet :
Sub rare et stable, espace profond, peu d'elements, queues controlees.

Fonction :
Pression par profondeur et vide.

Parametres critiques :
headroom, absence de boue, separation sub/halo, flow.

Risque :
Dub/reggae ou bass music trop identifiable.

Garde-fou :
Espace comme pression active, pas effet de style.
```

### 18.7. Jazz : voicing de partiels sans accord jazz

```text
Contexte :
Besoin d'une tension harmonique subtile.

Objet :
Trois resonances de gong/voix/bol placent un champ de partiels avec frottement lent.

Fonction :
Harmonie comme matiere.

Parametres critiques :
centre spectral, battements, resolution ou suspension, densite.

Risque :
Accord jazz plaque ou abstraction froide.

Garde-fou :
Relier la tension a l'espace, au corps ou au retour de scene.
```

### 18.8. Trance : halo vocal qui s'ouvre

```text
Contexte :
Expansion longue vers un peak non brutal.

Objet :
Voyelle etiree, multipliee en choeur spectral, ouverte progressivement dans l'espace.

Fonction :
Suspension, expansion, peak harmonique.

Parametres critiques :
ouverture spectrale, largeur, tension harmonique, fondation fantome.

Risque :
Vocal trance ou pad euphorique cliche.

Garde-fou :
Voix comme matiere/halo, pas chanson.
```

### 18.9. Acid : ligne de formant dans le didgeridoo

```text
Contexte :
Ligne vivante sans imitation TB-303.

Objet :
Modulation de formant extraite ou imitee du didgeridoo, repetee en motif court.

Fonction :
Ligne de matiere, torsion, hypnose timbrale.

Parametres critiques :
glissement, resonance, accent, microtonalite, relation au grave.

Risque :
Squelch acid reconnaissable ou gadget de filtre.

Garde-fou :
Le timbre porte la ligne plus que les notes.
```

### 18.10. Nature / acoustique : insecte comme micro-hat

```text
Contexte :
Remplacer une micro-pulsation aigue.

Objet :
Grain d'insecte ou frottement tres court, reorganise en pulse.

Fonction :
Hat naturalise / precision organique.

Parametres critiques :
attaque, brillance non continue, periodicite, fatigue aigue.

Risque :
Field recording decoratif ou tapis brillant.

Garde-fou :
La source sert une fonction rythmique, pas une image de nature.
```

### 18.11. Voix : syllabe-signal

```text
Contexte :
Marquer une transition sans lead vocal.

Objet :
Syllabe courte, peu intelligible, placee comme signal rythmique.

Fonction :
Accent, appel, changement de scene.

Parametres critiques :
intelligibilite, attaque consonantique, espace, repetition rare.

Risque :
Hook pop, gimmick vocal, narration.

Garde-fou :
Definir si la voix est signal, presence, rythme, texture ou sens.
```

### 18.12. Microtonalite : battement sub/halo

```text
Contexte :
Creer tension fine entre grave et espace.

Objet :
Sub stable avec halo legerement detune, battement lent perceptible.

Fonction :
Tension microtonale et lien sub/halo.

Parametres critiques :
vitesse du battement, niveau du halo, stabilite du sub, fatigue.

Risque :
Exercice microtonal ou grave flou.

Garde-fou :
Le battement doit produire attente, pression ou resolution.
```

### 18.13. Didgeridoo : grave partage

```text
Contexte :
Didgeridoo seul avec PC, presence libre mais compatible avec basse.

Objet :
Le didgeridoo porte les partiels et la rugosite ; le systeme ajoute une masse grave rare sous lui.

Fonction :
Grave partage, corps, profondeur.

Parametres critiques :
charge grave du didgeridoo, moment du sub, masquage, saillance.

Risque :
Didgeridoo reduit a basse ou masque par le systeme.

Garde-fou :
Mode DID-3 ou DID-4, role optionnel et reversible.
```

### 18.14. Gong : rugissement de retour

```text
Contexte :
Retour de masse apres suspension.

Objet :
Gong grave frotte puis frappe, queue sculptee et coupee avant flou.

Fonction :
Rugissement, drop rare, resolution spectrale.

Parametres critiques :
duree de preparation, densite de partiels, coupe du halo, relation au sub.

Risque :
Trailer, new age, halo envahissant.

Garde-fou :
Choisir : impact, halo, rugissement ou signal ; pas tout en meme temps.
```

### 18.15. Tambours / peaux : roll de build naturalise

```text
Contexte :
Build-up sans snare roll EDM.

Objet :
Frottements et peaux serrees qui augmentent en densite, avec attaques non uniformes.

Fonction :
Montee de tension, contraction rythmique.

Parametres critiques :
densite, registre medium, articulation principale, sortie.

Risque :
Percussion tribale ou roll EDM deguise.

Garde-fou :
Build par matiere et pression, pas par reflexe de style.
```

### 18.16. Transgression : cri, masse, silence

```text
Contexte :
Pic prepare dans une scene longue.

Objet :
Voix brute ou cri, soutenu par masse grave courte, puis silence ou retrait net.

Fonction :
Transgression T-3, rupture, reorientation de la scene.

Parametres critiques :
rarete, duree, saturation, sortie, fatigue.

Risque :
Agression gratuite ou melodrame.

Garde-fou :
Le pic doit avoir preparation et consequence.
```

### 18.17. Scene coherente complete : techno suspendue avec retour

```text
Depart :
Didgeridoo libre, halo de gong tres controle, souffle vocal discret.

Pression :
Pulse implicite par respiration, espace profond, centre grave latent.

Evolution :
Micro-pulses de bois apparaissent, puis accent fantome, puis peau profonde.

Retour :
Impact-fondation naturalise, sans kick electronique.

Inspirations actives :
Techno, sound system, trance naturalisee, microtonalite, didgeridoo.

Risques :
Ambient, boue grave, didgeridoo masque.

Garde-fous :
Memoire rythmique, monopole du sub, halo controle, retour de cadre.
```

### 18.18. Scene caricaturale a eviter

```text
Depart :
Kick techno standard, congas, wobble bass, supersaw, vocal chop, grande reverb.

Problemes :
Chaque inspiration apparait comme style reconnaissable.
Les sons electroniques sont identifiables.
Les fonctions se concurrencent.
La voix devient gimmick.
La naturalisation est absente.

Conclusion :
Ce n'est pas Vesperare ; c'est un collage de references.
```
